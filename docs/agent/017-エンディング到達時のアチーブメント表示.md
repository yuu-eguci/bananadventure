エンディング到達時のアチーブメント表示
===

## タスク概要
- `scene 14` または `scene 15` 到達時に、アチーブメント表示 UI を出す。
- 表示対象は `トゥルーエンド` `バッドエンド` `全アイテムコンプ` の 3 つ。
- 取得済み / 未取得を同時表示して見分けられる状態にする。

## 今回の要望 (統合者まとめ)
- エンディング到達時に、味気ない単文ではなく UI としてアチーブメントを表示したい。
- 取得できたものとできなかったものを、同じ画面内で判別可能にしたい。

## 変更対象
- `docs/agent/017-エンディング到達時のアチーブメント表示.md`
- `webapp/frontend-react/src/pages/HomePage.tsx`

## 詳細設計
### 表示条件
- アチーブメントカードは `scene.id === 14` または `scene.id === 15` のときに表示する。
- それ以外のシーンでは表示しない。

### 判定条件
- `トゥルーエンド`: 現在シーンが `scene 14` のとき取得済み。
- `バッドエンド`: 現在シーンが `scene 15` のとき取得済み。
- `全アイテムコンプ`: 現在プレイヤー所持アイテム ID が、`scenes.json` 内で取得可能な全アイテム ID をすべて含むとき取得済み。

### UI デザイン方針
- 右カラムへ `エンディングアチーブメント` カードを追加する。
- カード内に 3 行 (3 アチーブメント) を常時表示する。
- 各行は以下の要素で構成する。
  - 左: アチーブメント名 + 補足説明 (判定条件)
  - 右: `取得済み` または `未取得` のステータスバッジ
- 取得済み行は緑系の背景/枠、未取得行はグレー系の背景/枠で視覚的に区別する。

### 実装方針
- `HomePage.tsx` で `sceneData` を読み、取得可能アイテム ID の集合をモジュールスコープで作る。
- `viewModel` から現在シーン ID とプレイヤー所持アイテムを使って、3 アチーブメントの配列を生成する。
- 生成した配列を map で描画し、3 項目すべてを常時表示する。

### 期待効果
- エンディング時の達成状況が一画面で明確に把握できる。
- 「取れたもの / 取れなかったもの」の判別が即時にできる。
