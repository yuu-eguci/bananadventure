# BGM 追加フィージビリティレポート

## タスク概要

React アプリに BGM を追加したい。
ユーザのフィードバック: 「BGM 追加希望 (流れる流れないの2通りじゃなくて、ユーザがポチッとしたら流れる、もう一回ポチッとしたら止まるみたいな)」

### 今回の目的

- React アプリへの BGM 追加が技術的に可能かどうかを調査する
- 可能であれば、どのように実装するかの方針をまとめる
- 実装は React (Vite + TypeScript + MUI) 上で行う

### 求められる UI 挙動

- ボタンを押すと BGM が流れ始める
- もう一度ボタンを押すと BGM が止まる (トグル)
- ループ再生 (ゲーム BGM なので無限ループが望ましい)

### 注意事項

- React への音楽追加は今回が初の試み
- まずはフィージビリティと実装アプローチのレポートを作成する段階

## 詳細設計（プランナー記入欄）
### 1. React で BGM を扱う技術的な可否
結論、**技術的に十分可能**です。  
今回ほしい挙動（ボタンで再生/停止トグル、ループ再生）は、React の `useState` + `useRef` とブラウザ標準 API で実現できます。

- トグル: `isPlaying` 状態で制御
- 再生/停止: `audio.play()` / `audio.pause()`
- ループ: `audio.loop = true`
- 音量: `audio.volume = 0.2` などで調整

注意点は Autoplay Policy だけなので、**ユーザ操作起点で再生開始**にすれば実運用できます。

### 2. 利用可能なアプローチ比較
| アプローチ | 向いているケース | メリット | デメリット |
|---|---|---|---|
| `HTMLAudioElement` (`new Audio()`) | BGM 1〜2本、シンプルトグル | 実装が最短、学習コスト低い、依存追加なし | 高度なミキシングやエフェクトは弱い |
| Web Audio API (`AudioContext`) | 複数音源、フェード、ミキシング | 音制御が柔軟（GainNode など） | 実装コスト高、Autoplay 対応も少し複雑 |
| 音声ライブラリ（例: howler.js） | 将来 SE/BGM 管理を拡張したい | クロスブラウザ差分を吸収しやすい | 依存追加、プロジェクト規模に対して過剰になりやすい |

### 3. 推奨アプローチと理由
今回は **`HTMLAudioElement` を推奨**します。

- 要件が「BGM の ON/OFF トグル + ループ」に限定されている
- 依存を増やさずに短く安全に導入できる
- 既存構成（React + TypeScript + MUI）に自然に乗る
- 将来複雑化した時だけ Web Audio API / ライブラリへ段階移行しやすい

つまり、最初は軽く始めて、必要になったら拡張する方針がちょうど良いです。

### 4. 推奨アプローチの実装イメージ（コードスケッチ）
`useBgmPlayer` フックで再生制御を 1 箇所にまとめる想定です。

```tsx
// src/hooks/useBgmPlayer.ts
import { useEffect, useRef, useState } from "react";

export function useBgmPlayer(src: string) {
  const audioRef = useRef<HTMLAudioElement | null>(null);
  const [isPlaying, setIsPlaying] = useState(false);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const audio = new Audio(src);
    audio.loop = true;
    audio.volume = 0.25;
    audioRef.current = audio;

    return () => {
      audio.pause();
      audio.currentTime = 0;
      audioRef.current = null;
    };
  }, [src]);

  const toggle = async () => {
    if (!audioRef.current) return;
    try {
      if (isPlaying) {
        audioRef.current.pause();
        setIsPlaying(false);
        return;
      }
      await audioRef.current.play(); // ユーザ操作起点で呼ぶ
      setIsPlaying(true);
      setError(null);
    } catch {
      setError("音声再生がブラウザにブロックされました");
    }
  };

  return { isPlaying, toggle, error };
}
```

`HomePage` ではこんな感じでトグルボタンを置きます。

```tsx
const { isPlaying, toggle } = useBgmPlayer("/bgm/adventure-loop.mp3");

<Button variant="contained" onClick={toggle}>
  {isPlaying ? "BGM OFF" : "BGM ON"}
</Button>
```

### 5. 音声ファイルの配置場所
このプロジェクトは画像を `src/assets` と `public` の両方で扱っていますが、BGM はまず **`public/bgm/` 配置**を推奨します。

- 推奨配置: `webapp/frontend-react/public/bgm/adventure-loop.mp3`
- 参照方法: `"/bgm/adventure-loop.mp3"`

理由:
- `assetImageResolver` は画像専用の解決ロジックで、音声は対象外
- BGM はビルド時 import なしで固定パス参照できると運用がシンプル
- 音源差し替え時にコード変更が最小になる

### 6. ブラウザ自動再生ポリシー（Autoplay Policy）対応方針
基本方針は「**初回再生は必ずユーザ操作で開始**」です。

- 初期状態は `isPlaying = false`
- `useEffect` で自動 `play()` はしない
- ボタンクリック内で `play()` を呼ぶ
- `play()` の Promise reject（`NotAllowedError` 等）を捕捉して UI に案内を出す
- モバイル Safari 含め、ユーザ操作起点なら通る設計にする
