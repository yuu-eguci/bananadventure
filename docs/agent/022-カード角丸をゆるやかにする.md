# カード角丸をゆるやかにする設計ノート

## タスク概要
- 目的: 各画面のカード UI の角丸 (R) を、現在よりゆるやかにして「ばななちゃんの丸っこいイメージ」を反映する
- 背景: ユーザ feedback に基づく UI 微調整

## 今回の要望（原文要約）
- 各画面、カードの角のエッジ (R) をゆるやかにしたい
- 細かい見た目調整だが、脚本担当の「丸っこいイメージ」を反映したい

## 詳細設計（プランナー記入欄）
- 対象画面
  - `webapp/frontend-react/src/pages/HomePage.tsx`（現状カード UI があるメイン画面）
- 対象カード/パネル
  - シーン表示エリアの外枠 `Paper`
  - シーン内の選択肢 `Card`
  - エンディングアチーブメントの親 `Card`
  - アチーブメント一覧の各行（カード見た目の `Box`）
  - バナナメーター `Card`
  - インベントリアイテム `Card`
- 角丸の変更方針
  - 角丸は「ほんの少し」ではなく、見て分かる程度にやわらかくする
  - ただし丸すぎてポップすぎる印象にならないよう、画面内で 2〜3 段階に統一する
  - 目安値（px）
    - 大きい面（シーン表示 `Paper`）: 18
    - 通常カード（選択肢/メーター/インベントリ/アチーブメント親）: 14
    - 内部の小パネル（アチーブメント各行）: 12
  - ステータスのピル（`borderRadius: 999`）は「カード角丸」対象外なので現状維持
- 確認方法
  - PC 幅（`md` 以上）とスマホ幅（`xs`）で、各カードの角丸が揃って見えるかを目視確認
  - シーン画像 + オーバーレイ + カードの重なりで角の欠け/はみ出しが出ないか確認
  - 選択肢カード、インベントリカードのクリック領域や操作感が変更前と同じか確認
  - 差分レビュー時は「角丸のみの変更」に寄っているか（余計な色/余白変更がないか）を確認

## 実装メモ（プランナー記入欄）
- ※今回は設計のみ。ここは実装予定手順のメモ
- 予定手順
  1. `HomePage.tsx` 内で角丸対象コンポーネントを洗い出し、`sx.borderRadius` の現状値を整理する
  2. 角丸値を上記方針（18 / 14 / 12）へ反映する（対象外のピルは変更しない）
  3. 画面確認を実施し、PC/スマホで視覚バランスと操作性をチェックする
  4. 必要なら 1 段階だけ微調整し、最終値をこのノートへ追記する
  5. 変更差分をセルフレビューし、角丸以外の意図しない変更がないことを確認する
- 実装フェーズ追記（実績）
  - `HomePage.tsx` の対象コンポーネントに `borderRadius` を追加/更新
    - シーン表示エリア外枠 `Paper`: `18px`
    - 通常カード（選択肢 / エンディングアチーブメント親 / バナナメーター / インベントリ）: `14px`
    - アチーブメント各行（内部小パネル）: `12px`
  - ステータスピル（`borderRadius: 999`）は変更なし
  - 最終値の微調整結果: 追加の微調整なし（設計値 `18 / 14 / 12` をそのまま採用）
  - 差分は角丸調整のみに限定（色・余白・レイアウトは非変更）

## オーナー向け要約
- 対象は `HomePage.tsx` の主要カード UI（シーン枠、選択肢、アチーブメント、バナナメーター、インベントリ）
- 角丸は `18 / 14 / 12` の 3 段階で統一し、丸っこさを出しつつ過度にポップにならない設計
- `borderRadius: 999` のステータスピルは対象外として現状維持
- 確認観点はレスポンシブ表示、重なり時の見え方、クリック領域維持、角丸以外の差分混入防止
