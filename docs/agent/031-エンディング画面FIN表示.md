# 031 - エンディング画面 FIN 表示

## 今回の要望

エンディング画面に "FIN" 的な表示がほしい。終わったのか分かりづらいので。

エンディングのシーンでは、わかりやすくしたいね。シーンメッセージの右下とかに "FIN" だか "おしまい" だか "アチーブメントを確認してね" だか、出せばいいかな。

## 背景・現状

- `HomePage.tsx` には `isEndingScene` という boolean がすでにある
  - `currentSceneId === ENDING_SCENE_IDS.TRUE || currentSceneId === ENDING_SCENE_IDS.BAD`
  - `ENDING_SCENE_IDS.TRUE = 14`, `ENDING_SCENE_IDS.BAD = 15`
- エンディングに到達すると右カラムにアチーブメントカードが表示される
- シーンメッセージエリア (左カラム) には特に "終わった" を示す要素がない

## 詳細設計

### 変更のゴール

1. エンディングシーン到達時だけ、シーンメッセージエリアの右下に FIN 表示を出す
2. FIN 表示だけでなく、次の行動がわかる補助文言（アチーブメント確認）も添える
3. 既存の選択肢 UI、アチーブメントカード、ジングル演出の挙動は変えない

### UI 方針

- 表示条件は既存の `isEndingScene` をそのまま使う
- 表示位置は左カラム下部のメッセージオーバーレイ内、右寄せ
- 文言は以下を採用
  - メイン: `FIN`
  - サブ: `アチーブメントを確認してね`
- 見た目はメッセージ領域背景の上でも読めるように、半透明背景 + 白文字 + 丸み付き

### 実装方針

- 変更対象: `webapp/frontend-react/src/pages/HomePage.tsx`
- 既存のメッセージ表示用 `Box` の末尾に、`isEndingScene` 条件で FIN バッジを追加
- 既存レイアウトへの影響を最小化するため、新しい state は追加しない
- `isEndingScene` 判定やアチーブメント判定ロジックは変更しない

### 受け入れ条件

- scene 14 / 15 でのみ FIN 表示が出る
- エンディング以外のシーンでは FIN 表示が出ない
- 選択肢表示、インベントリ、アチーブメントカードの既存機能が壊れない
- 見た目として、終端到達と次の確認行動が即座に伝わる

#### 実装コードのレビュー

1. **定数定義 (L37-38)**
   ```tsx
   const endingSceneLabel = "FIN";
   const endingSceneHint = "アチーブメントを確認してね";
   ```
   - 文言をハードコードせず定数化している点が良い
   - ただし、これらは他の定数 (`ENDING_SCENE_IDS` など) と同じブロックに配置されているが、命名規則が異なる (camelCase vs SCREAMING_SNAKE_CASE)
   - **軽微な指摘**: 命名規則を統一するなら `ENDING_SCENE_LABEL` / `ENDING_SCENE_HINT` にするか、あるいは既存の定数も camelCase にするかを検討してもよい (必須ではない)

2. **条件分岐とレイアウト (L343-379)**
   ```tsx
   {isEndingScene ? (
     <Box sx={{ mt: 1.25, display: "flex", justifyContent: "flex-end" }}>
       ...
     </Box>
   ) : null}
   ```
   - `isEndingScene` が true のときだけ表示する条件分岐が適切に配置されている
   - `justifyContent: "flex-end"` で右寄せが実現されている
   - バッジの内部スタイル (`bgcolor`, `border`, `borderRadius`) も設計書の「半透明背景 + 白文字 + 丸み付き」を満たしている

3. **視認性とデザイン**
   - `bgcolor: "rgba(0,0,0,0.3)"` で半透明背景を実現
   - `border: "1px solid rgba(255,255,255,0.45)"` で境界線を引いている点も良い
   - `fontWeight: 800` と `letterSpacing: "0.12em"` で FIN を強調している点も視認性向上に貢献

4. **アクセシビリティ**
   - セマンティックに `Typography` コンポーネントを使っている点は良い
   - ただし、エンディングに到達したことを伝える重要な情報なので、スクリーンリーダー対応を考えるなら `role="status"` や `aria-live="polite"` を検討してもよい (必須ではない)
