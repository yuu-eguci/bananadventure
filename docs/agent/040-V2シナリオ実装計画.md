# 040 - V2シナリオ実装計画

## 要望

- これまで `HomePageV2` に移植してきた UI（メインセクション、BGM ボタン、バナナメーター、アイテム、メッセージ／選択肢オーバーレイ）を使って、`HomePage`（V1）を見本にゲームのシナリオを V2 へ実装する
- 作業者が少しずつ触れるように、切りの良い単位（= 1 コミット相当）でまとめる
- 実装は引き続きコンポーネントをきちんと分ける方針を守る

## V1 と変える点（このノートの前提）

1. **選択肢レスポンステキストの扱い**
   - V1: 選択肢を押すと専用カード（`inlineMessage.kind === "scene"`、`タップして次へ`）で `responseText` を表示し、タップで次シーンへ進んでいた
   - V2: この専用カードは**廃止**。選択肢を押したら即座に次シーンへ進み、`responseText` は**次シーンのメッセージ（`scene.text`）の前**にリード文として表示する
2. **エンディング表示**（確認済み）
   - V1: scene 14/15 到達時に右カラムへアチーブメント表を表示
   - V2: **モーダル（Dialog）で表示**する。V2 の Dialog 統一方針（メインカラー・角丸 18px）に沿う
3. **アイテム使用後のフィードバック**（確認済み）
   - V1: インベントリ欄にインライン表示（`○○ を使用した！ばななメーター +N`）
   - V2: アイテムモーダル内で「使う」を押した後、**同じモーダル内で結果（メーター増減）を表示**してから閉じる
4. **シーン遷移演出**（確認済み）
   - V1 と同じく、初期化時・シーン遷移時にローディング GIF（jingle backdrop）を全画面表示する

## 設計方針

- ゲーム状態（`SceneViewModel` の保持、初期化、選択肢選択、アイテム使用、リセット、loading 状態）は **カスタムフック `useHomePageV2Game` に集約**する
  - V1 では `HomePage.tsx`（671 行）に状態とロジックが密集していた。V2 ではページを薄く保ち、フックとプレゼンテーション用コンポーネントに分ける
  - `SceneService` はそのまま再利用（ロジックは確立済み）
- `responseText` のリード文は、フックが「直前に選んだ選択肢の `responseText`」を状態として保持し、次シーンのオーバーレイ先頭に渡す
- アイテム欄はスタブ定数（`stubItem1/2/3`）をやめ、`player.items`（進行中に入手した実アイテム）を描画する
  - これは「ゲーム進行中に入手したら出す」仕様そのもの。実データ配列の描画なので `map` で問題ない（スタブのときに `map` を避けたのは「静的な一覧の先取り表示」を避ける意図だったため）

## 作業単位（各まとまり = 1 コミット想定）

### A. ゲーム状態フック `useHomePageV2Game` の新設

- `webapp/frontend-react/src/hooks/useHomePageV2Game.ts` を新規作成
- 保持する状態: `viewModel`, `isLoading`(jingle 用), `leadResponseText`（次シーンに表示するリード文）
- 公開する API（案）: `scene`, `player`, `isLoading`, `leadResponseText`, `selectChoice(choice)`, `useItem(item)`, `reset()`
- マウント時に `SceneService.fetchInitialViewModel()` で scene 0 を初期化
- まだ UI には繋がない。フックだけ用意して `yarn build` が通ることを確認
- **確認できること**: 型・ビルドが通る（画面の見た目は変わらない）

### B. メッセージ・選択肢オーバーレイの本実装（`SceneOverlayPreview` → 実コンポーネント化）

- `SceneOverlayPreview.tsx` のスタブ（固定文言・固定選択肢）を実データ描画に置き換える（必要ならファイル名を `SceneOverlay.tsx` にリネーム）
- `scene.text` と `scene.sceneChoices` を `viewModel` から描画
- 選択肢タップ → `useHomePageV2Game.selectChoice(choice)` を呼ぶ
- `leadResponseText` があれば `scene.text` の**前**にリード文ブロックとして表示（V1 のタップカードは作らない）
- `HomePageV2.tsx` をフック接続に切り替え（`stubItem` 群はこの時点ではアイテム欄に残してよいが、メッセージ／選択肢は実データへ）
- **確認できること**: タイトル「はじめる？」から選択肢で実際にシーンが進み、選んだ選択肢のレスポンスが次シーンの先頭に出る

### C. ローディング jingle backdrop の移植

- V1 の `Backdrop` + `loading-00001.gif` を V2 に移植
- 初期化時とシーン遷移時に `isLoading` で表示
- **確認できること**: シーン遷移のたびにローディング GIF が出る（V1 と同じ体験）

### D. アイテム入手・使用の接続

- アイテム欄を `stubItem1/2/3` から `player.items`（入手済みアイテム）描画へ切り替え
- `ItemWidget` のモーダルで「使う」→ `useHomePageV2Game.useItem(item)` を実行
- 使用後、同じモーダル内に結果（`ばななメーター +N` / `-N`）を表示してから閉じる導線にする
- 使用済みアイテムは `disabled`（既存の `opacity 0.4` + `elevation 0`）
- **確認できること**: 選択肢で入手したアイテムが右パネルに出て、モーダルから使用でき、結果が出る

### E. バナナメーター連動・ゲームオーバー・トリガー確認

- `BananaMeterWidget` の `value` を `player.bananaMeter` に接続
- バナナメーター ≤ 0 → ゲームオーバー（scene 15）への遷移は `SceneService` 側で処理済みなので、フック経由で正しく反映されることを確認
- `triggerItems`（アイテム所持によるシーン分岐）も `SceneService` 側で処理済み。実プレイで通ることを確認
- **確認できること**: メーターが UI に反映され、0 でゲームオーバーへ、トリガーアイテムで分岐する

### F. エンディング演出（アチーブメントモーダル）

- scene 14（トゥルーエンド）/ 15（バッドエンド）到達時にアチーブメントモーダル（Dialog）を表示
- 表示内容は V1 同様: トゥルーエンド到達 / バッドエンド到達 / 全アイテムコンプ（`collectedItemCount / allCollectibleItemIds.length`）
- Dialog はメインカラー・角丸 18px の V2 統一方針に合わせる
- アチーブメント判定ロジックは V1 から移植（`ENDING_SCENE_IDS`, `allCollectibleItemIds` の算出）
- **確認できること**: エンディング到達でアチーブメントモーダルが出る

### G. リセット導線

- V2 にもリセット導線を用意し、`useHomePageV2Game.reset()` で scene 0 へ戻す（`leadResponseText` もクリア）
- 配置は V1 の `ResetButton` を参考に、V2 のレイアウトに合う形で検討
- **確認できること**: リセットで最初からやり直せる

## 補足・確認済み事項

- BGM のトラック切り替え（通常 `main` / エンディング `ending`）: V1 は `isEndingScene` で `trackKey` を切り替えていた。V2 でもエンディング演出（F）の段で同様に対応する想定
- 状態管理はフックに集約し、`HomePageV2.tsx` は配線（フックと各コンポーネントの接続）に徹する
