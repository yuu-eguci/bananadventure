# 032 - 選択肢表示中はタップ次へラベル非表示

## 要望

- 選択肢が表示されているときに "タップして次へ" というラベルが出ているけど、このタイミングでは不要なので非表示にしたい
- 選択肢が消えた分、タップ領域を大きくして UX を向上させたい

## 詳細設計

### 変更のゴール

1. 選択肢が表示されている状態では、`タップして次へ` ラベルを非表示にする
2. ラベルを消しても操作しやすさが落ちないよう、該当カードのタップ領域を拡大する
3. 既存のシーン遷移ロジック (`advanceScene`) と表示切り替えロジックは壊さない

### 背景と現状整理

- `HomePage.tsx` では `タップして次へ` が 2 箇所ある
  - `inlineMessage.kind === "scene"` のシーン返信カード
  - `inlineMessage.kind === "item"` のアイテム使用メッセージカード
- 一方で「選択肢表示中」の状態は `hasSceneChoices === true` で判定できる
- シーン返信カードは `inlineMessage.kind === "scene"` のときだけ表示され、この分岐中は選択肢 UI 自体が出ない
- なので今回の主対象は「アイテム使用メッセージカードのタップヒント」。ここだけ `hasSceneChoices` と連動させるのが最小差分で安全

### UI 制御方針（表示条件を明文化）

- `HomePage.tsx` に以下の判定値を追加する
  - `const shouldShowSceneTapHint = inlineMessage?.kind === "scene";`
  - `const shouldShowItemTapHint = inlineMessage?.kind === "item" && !hasSceneChoices;`
- 表示ルールは次の 3 状態で固定する
  - `inlineMessage.kind === "scene"`: シーン返信カード内の `タップして次へ` を表示する
  - `inlineMessage.kind === "item" && hasSceneChoices === true`: アイテムカードの `タップして次へ` は非表示
  - `inlineMessage.kind === "item" && hasSceneChoices === false`: アイテムカードの `タップして次へ` を表示する
- これで「選択肢が見えているのに次へラベルが出る」違和感だけを狙って解消し、既存のシーン返信 UX は維持する

### タップ領域拡大方針（スタイル値まで確定）

- 対象は `inlineMessage.kind === "item"` のカードだけに限定する（他カードは現状維持）
- `CardActionArea` に `sx` を追加し、`hasSceneChoices` が `true` のときに高さを増やす
  - `minHeight: { xs: 132, sm: 148 }`（選択肢表示中）
  - `minHeight: { xs: 112, sm: 128 }`（それ以外）
- `CardContent` も `py` を連動させる
  - `py: { xs: 2.5, sm: 3 }`（選択肢表示中）
  - `py: { xs: 2, sm: 2.5 }`（それ以外）
- これでラベルを消したときでも「どこを押せば閉じるか」が狭くならないようにする

### 実装対象

- `webapp/frontend-react/src/pages/HomePage.tsx`
  - タップヒント表示の判定値（`shouldShowSceneTapHint`, `shouldShowItemTapHint`）を追加
  - シーン返信カードとアイテム使用メッセージカードのヒント描画条件を分離
  - アイテム使用メッセージカードの `CardActionArea` / `CardContent` に `hasSceneChoices` 連動のサイズ指定を追加

### 実装ステップ

1. `hasSceneChoices` 定義の直後に、タップヒント制御用の 2 つの定数を追加する
2. シーン返信カード側のヒント描画を `shouldShowSceneTapHint` でラップする
3. アイテム使用メッセージカード側のヒント描画を `shouldShowItemTapHint` でラップする
4. 同カードの `CardActionArea` / `CardContent` に `hasSceneChoices` 連動の `minHeight` / `py` を設定する
5. 手動確認で 3 状態（scene inline / item inline + choicesあり / item inline + choicesなし）を順に確認する

### 受け入れ条件

- 選択肢が表示中のとき、`タップして次へ` ラベルは表示されない
- 選択肢が表示されていない遷移待ち状態では、`タップして次へ` ラベルが従来どおり表示される
- ラベル非表示後も、該当カードのタップ操作がしやすい高さを維持できる
- 既存のシーン遷移、アイテム使用、エンディング表示の挙動に副作用がない

## オーナー向け要約

- 表示条件を 2 つの判定値に分けて整理する
  - `shouldShowSceneTapHint`: シーン返信カードでは従来どおりラベル表示
  - `shouldShowItemTapHint`: アイテムカードは「選択肢なし」のときだけラベル表示
- つまり「選択肢が見えている間だけ、アイテムカードの `タップして次へ` を隠す」仕様になる
- ラベル非表示中でも押しやすいように、アイテムカードのタップ領域を `minHeight` / `py` で拡大する
  - 選択肢表示中: `minHeight xs132/sm148`, `py xs2.5/sm3`
  - それ以外: `minHeight xs112/sm128`, `py xs2/sm2.5`
- 変更対象は `HomePage.tsx` の表示ロジックとスタイルのみで、`advanceScene` などの遷移処理には手を入れない
