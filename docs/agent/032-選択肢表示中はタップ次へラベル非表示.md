# 032 - 選択肢表示中はタップ次へラベル非表示

## 要望

- 選択肢が表示されているときに "タップして次へ" というラベルが出ているけど、このタイミングでは不要なので非表示にしたい
- 選択肢が消えた分、タップ領域を大きくして UX を向上させたい

## 詳細設計

### 変更のゴール

1. 選択肢が表示されている状態では、`タップして次へ` ラベルを非表示にする
2. ラベルを消しても操作しやすさが落ちないよう、該当カードのタップ領域を拡大する
3. 既存のシーン遷移ロジック (`advanceScene`) と表示切り替えロジックは壊さない

### 背景と現状整理

- `HomePage.tsx` では `タップして次へ` が 2 箇所ある
  - `inlineMessage.kind === "scene"` のシーン返信カード
  - `inlineMessage.kind === "item"` のアイテム使用メッセージカード
- 一方で「選択肢表示中」の状態は `hasSceneChoices === true` で判定できる
- 今回の要望は「選択肢表示中のラベル非表示」なので、表示条件を `hasSceneChoices` に連動させるのが最小差分で安全

### UI 制御方針

- `HomePage.tsx` に「タップヒントを出すか」の判定値を追加する
  - 例: `const shouldShowTapHint = !hasSceneChoices || inlineMessage?.kind === "scene";`
- この判定を `タップして次へ` を描画している 2 箇所に適用する
  - シーン返信カード: 従来どおり表示（選択肢 UI が出ない状態のため）
  - アイテム使用メッセージカード: `hasSceneChoices === true` の間は非表示
- これで「選択肢が見えているのに次へ誘導ラベルが出る」違和感をなくす

### タップ領域拡大方針

- ラベル非表示の対象になるアイテム使用メッセージカード側で、`CardActionArea` の最小高さを引き上げる
  - 目安: `minHeight` を `xs/sm` で段階設定（スマホ優先で大きめ）
- 合わせて `CardContent` の縦方向余白 (`py`) を増やし、視覚的にも押せる領域を明確化する
- 文言量に依存せず一定サイズを確保できる構成にする

### 実装対象

- `webapp/frontend-react/src/pages/HomePage.tsx`
  - タップヒント表示条件の判定値を追加
  - `タップして次へ` ラベル描画の条件分岐を更新
  - アイテム使用メッセージカードの `CardActionArea` / `CardContent` サイズ調整

### 受け入れ条件

- 選択肢が表示中のとき、`タップして次へ` ラベルは表示されない
- 選択肢が表示されていない遷移待ち状態では、`タップして次へ` ラベルが従来どおり表示される
- ラベル非表示後も、該当カードのタップ操作がしやすい高さを維持できる
- 既存のシーン遷移、アイテム使用、エンディング表示の挙動に副作用がない

- 受け入れ条件を「表示条件」「操作性」「副作用なし」の 3 点で確認できる形にした
