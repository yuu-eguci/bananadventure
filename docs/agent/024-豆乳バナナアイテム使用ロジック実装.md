# 豆乳バナナアイテム使用ロジック実装

## タスク概要
アイテム「豆乳バナナ」に関するバグを修正する。

### 企画の設定
- アイテム「豆乳バナナ」をボタンで使用すると、ばななメーターが +20 回復する
- シーン「豆乳バナナが腐って豆乳ヨーグルトに！」(scene 11) は、豆乳バナナを**未使用**で持っている場合に発生する罠シーン
  - 強制的にばななメーターが -50 される
  - 早めに使っておくと、この罠シーンを回避できる

### 現状のバグ
1. `onPressItem` が `console.log(item)` だけで、アイテム使用ロジックが未実装
2. `SceneService` にアイテム使用メソッドがない
3. `SceneService` の `triggerItems` チェックが `used` フラグを無視している
   - 使用済みアイテムでも罠シーンに飛んでしまう

### 修正対象ファイル
- `webapp/frontend-react/src/services/SceneService.ts`
- `webapp/frontend-react/src/pages/HomePage.tsx`

## 詳細設計（プランナー記入欄）
### 変更のゴール
- `インベントリのアイテムボタン押下` で、アイテム効果を実際に反映する
- 豆乳バナナ（item id: `0`）を使用済みにした場合、`scene 10` の `triggerItems` で `scene 11` へ遷移しないようにする
- 既存のシーン選択フローを壊さず、差分は `SceneService` と `HomePage` の最小範囲にとどめる

### 実装対象と責務
- `webapp/frontend-react/src/services/SceneService.ts`
  - アイテム使用のユースケースを 1 メソッドに集約する
  - `triggerItems` 判定を「所持している」ではなく「未使用で所持している」に変更する
- `webapp/frontend-react/src/pages/HomePage.tsx`
  - `onPressItem` から `SceneService` のアイテム使用メソッドを呼び出す
  - 返ってきた `SceneViewModel` で画面状態を更新する

### SceneService の詳細設計
- 追加メソッド
  - `useItem({ viewModel, itemId }): Promise<SceneViewModel>` を追加する
  - 入力: 現在の `viewModel` と押下されたアイテム ID
  - 出力: アイテム効果とトリガー判定反映後の `SceneViewModel`
- `useItem` の処理手順
  1. `viewModel.player.items` から対象アイテムを検索する
  2. 未所持または `used === true` の場合は、状態を変えずそのまま返す（ガード）
  3. `player` と `items` を不変更新で複製し、対象アイテムのみ `used: true` にする
  4. `bananaMeterDelta` を加算し、変化があれば `bananaMeterChanged = true` を立てる
  5. `itemsChanged = true` を立てる
  6. `bananaMeter <= 0` の場合は既存仕様どおり `GAMEOVER(15)` を優先する
  7. それ以外は現在シーン起点で `triggerItems` を再評価し、必要なら遷移先シーンへ進める
- `triggerItems` 判定ロジック修正
  - 既存の `hasItem = items.some((i) => i.id === item.id)` を廃止
  - 新条件を `hasUnusedItem = items.some((i) => i.id === item.id && !i.used)` に変更
  - `selectSceneChoice` 内の判定にも同じ条件を適用し、シーン選択時・アイテム使用時で挙動を統一する
- 内部共通化
  - `triggerItems` の `while` ループは private メソッド化（例: `resolveTriggeredScene`）して、`selectSceneChoice` / `useItem` の両方から使える形にする
  - 目的は重複排除と条件不一致の防止

### HomePage の詳細設計
- 型の明確化
  - `onPressItem` の引数型を `object` から `Item` に変更する
- 押下時フロー
  1. `viewModel` が `null` なら何もしない
  2. `service.useItem({ viewModel, itemId: item.id })` を await
  3. 戻り値を `setViewModel` へ反映
- UI 連動
  - 既存の `item.used` による `disabled` / 半透明表示はそのまま活かす
  - これにより、使用後はボタンが即時無効化され、再使用を防げる

### 期待される挙動（今回タスクの受け入れ条件）
- 豆乳バナナ未使用で `scene 10` に入ると、従来どおり `scene 11`（罠）へ遷移する
- 豆乳バナナを事前に使用すると、`scene 10` で `scene 11` へ遷移しない
- 豆乳バナナ使用時にバナナメーターが `+20` される
- 使用済みアイテムは再度押せず、状態表示が `使用済み` になる
- 既存のゲームオーバー遷移（`bananaMeter <= 0` で `scene 15`）は維持される

### 非対象（今回やらないこと）
- アイテム演出 UI（効果音、トースト、確認ダイアログ）追加
- `bananaMeterDeltaPercent` の新ルール導入
- バックエンド API 連携への移行
