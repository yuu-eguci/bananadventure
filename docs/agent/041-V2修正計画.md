# 041 - V2 修正計画（タスク 20）

A〜G の実装でゲームは一通り動くようになったが、いくつか不具合・調整点がある。
以下、各修正を実装レベルで分解した。作業者はこれをそのまま着手できる。

> 補足: 修正 2 と修正 5 はどちらもレイアウト計測（`useHomePageV2SceneOverlayLayout` と `MainSection`）に手を入れるため、**修正 5 → 修正 2 の順**でまとめて実装するのがよい。ほかは独立して着手できる。

## 修正 1. V2 画面に V1 への遷移ボタンは不要

### 目的
`ResetButton` は V1/V2 共通コンポーネントで、`page="v2"` のとき `V1` へ戻るボタンを出している。V2 ではこの V1 遷移ボタンを消す。

### 対象ファイル
- `webapp/frontend-react/src/components/ResetButton.tsx`

### 実装手順
1. バージョン切り替えボタン（`versionLink` / `versionLabel` の `<Button>`）を、`page !== "v2"` のときだけ描画するようにする。
   - これで V1 側は従来どおり `V2` ボタンが出て、V2 側ではボタンが消える。
2. `page === "v2"` のときに使われなくなる `versionLink` / `versionLabel` は、条件分岐の中だけで使うか、不要なら整理する。
   - `設定資料集`（lore）ボタンと `Reset` ボタンはそのまま残す（今回の対象外）。

### 注意点
- V1（`HomePage`）側の `ResetButton` 利用箇所（`page` 未指定 = `"v1"`）の挙動は変えない。

## 修正 2. メッセージオーバーレイの top を「メインセクション下から N%」の定数で決める

### 目的
現状 `sceneOverlayTop` は「右上ボタン群の高さ + 24px」で決まっており、オーバーレイの頂点が高すぎて画像を覆ってしまう。
これを「**メインセクションの下から N% の高さぶんをオーバーレイが占める**」という定数ベースの指定に変える。

### 対象ファイル
- `webapp/frontend-react/src/hooks/useHomePageV2SceneOverlayLayout.ts`
- `webapp/frontend-react/src/components/HomePageV2/HomePageV2RightPanel.tsx`（`rightPanelRef` が不要になるため）
- `webapp/frontend-react/src/pages/HomePageV2.tsx`（`rightPanelRef` の受け渡し除去）

### 実装手順
1. フックに定数を追加する。意味は「オーバーレイがメインセクション画像エリアの下から占める高さの割合」。
   ```ts
   // オーバーレイは画像エリアの下から、この割合ぶんの高さを占める（例: 0.45 = 下から 45%）
   const SCENE_OVERLAY_HEIGHT_RATIO = 0.45;
   ```
2. `sceneOverlayTop` を、ボタン群の計測ではなく **画像エリアの高さ × (1 - ratio)** で求める。
   - 画像エリアの高さは修正 5 で動的計測する（`mainImageHeight`）。修正 5 と合わせて
     `sceneOverlayTop = Math.round(mainImageHeight * (1 - SCENE_OVERLAY_HEIGHT_RATIO))` とする。
   - （修正 5 を先にやらない場合の暫定: `MAIN_SECTION_HEIGHT` 定数 × (1 - ratio)。ただし最終形は計測値ベース。）
3. ボタン群の高さ計測 useLayoutEffect と `RIGHT_PANEL_TO_OVERLAY_GAP_PX`、`rightPanelRef` を削除する。
4. `HomePageV2RightPanel` は `forwardRef` をやめ、通常コンポーネントに戻す。`HomePageV2` 側も `ref={rightPanelRef}` を外す。

### 注意点（重要）
- **「オーバーレイの height が大きいときは見切れさせず、メインセクションの下にはみ出させる」仕様は必ずキープする。**
  これは `overlayExtensionHeight` の計算（`Math.max(0, sceneOverlayTop + overlayの実 scrollHeight - 画像エリア高さ)`）で実現されている。top の決め方を変えても、この延長ロジックはそのまま生かすこと。
- ratio の値（0.45 など）は見た目を見ながら調整可能。定数 1 箇所で調整できる形にしておくのが今回の主旨。

## 修正 5. 画像比率を 1:1 にする（スマホは高さ重視・横幅見切れ）

### 目的
シーン画像はすべて 1:1。現状は固定高さ × 横幅いっぱい（横長）で相性が悪い。
- **sm 以上（横幅が広い）**: メインセクション画像エリアを 1:1（正方形）にする。
- **xs（スマホ・横幅が小さい）**: これまでどおり高さ重視（固定高さ）、横幅は `objectFit: cover` で見切れて OK。

### 対象ファイル
- `webapp/frontend-react/src/components/HomePageV2/MainSection.tsx`
- `webapp/frontend-react/src/hooks/useHomePageV2SceneOverlayLayout.ts`（高さが動的になるため計測へ切り替え）
- `webapp/frontend-react/src/pages/HomePageV2.tsx`（計測用 ref の受け渡し）

### 実装手順
1. `MainSection` の画像エリア `Box` を、ブレークポイントで切り替える。
   ```tsx
   <Box
     sx={{
       position: "relative",
       width: "100%",
       // xs: 高さ重視（固定）。sm 以上: 1:1 正方形（高さは幅に追従）。
       height: { xs: MAIN_SECTION_HEIGHT.xs, sm: "auto" },
       aspectRatio: { xs: "auto", sm: "1 / 1" },
     }}
   >
   ```
   - 画像 `<Box component="img">` は従来どおり `width:100% / height:100% / objectFit:"cover"`。これで xs は高さ優先で横が見切れ、sm 以上は正方形に収まる。
2. 画像エリアの高さがブレークポイント・ウィンドウ幅で変わるため、レイアウトフックは **定数 `MAIN_SECTION_HEIGHT` ではなく実測値**を使うように変える。
   - 画像エリア `Box` に ref を付け、`ResizeObserver` で高さ（`getBoundingClientRect().height`）を計測して `mainImageHeight` として持つ。
   - この `mainImageHeight` を、修正 2 の `sceneOverlayTop` 計算と、`overlayExtensionHeight` 計算（`requiredHeight - mainImageHeight`）の両方に使う。
   - これにより `isMdUp` / `MAIN_SECTION_HEIGHT` ベースの分岐は不要になる（`MAIN_SECTION_HEIGHT.xs` は MainSection の xs 高さ指定としては残す）。
3. ref の配線:
   - フックが `mainImageRef` を返し、`HomePageV2` から `MainSection` の画像エリアへ渡す（`MainSection` に `imageAreaRef` prop を追加、または `forwardRef` 化）。

### 注意点
- 1:1 にすると、`maxWidth: 960` の広い画面では画像エリアが最大 960×960 と大きくなる。まずは指示どおり 1:1 で出してみて、大きすぎる場合は別途調整する想定。
- 計測は初回レンダーとリサイズの両方で更新されるよう `ResizeObserver` を使う（`useLayoutEffect` 内で生成・破棄）。
- 修正 2 の延長（はみ出し）ロジックは、`mainImageHeight` 実測ベースになっても同じ式で機能する。

## 修正 6. エンディングは自動でモーダルを出さず、FIN ボタン経由で出す

### 目的
エンディング到達時にアチーブメントモーダルが即座に開くと、直前のメッセージ（最後のシーン本文）が読めない。
最後のシーン（scene 14 / 15、いずれも選択肢ゼロを確認済み）では、**選択肢と同じ位置に「FIN｜アチーブメントを確認」ボタン**を出し、タップでアチーブメントモーダルを開く。

### 対象ファイル
- `webapp/frontend-react/src/pages/HomePageV2.tsx`
- `webapp/frontend-react/src/components/HomePageV2/SceneOverlay.tsx`

### 実装手順
1. `HomePageV2` の **自動オープン useEffect を削除**する（`isEndingScene` を見て `setIsEndingDialogOpen(true)` していた箇所）。
   - `isEndingDialogOpen` ステートは残す。開くのは FIN ボタン押下時のみ。
   - reset 時に閉じる挙動（`setIsEndingDialogOpen(false)`）はそのまま残す。
2. `SceneOverlay` に props を追加する: `isEndingScene: boolean` と `onOpenEnding: () => void`。
3. `SceneOverlay` の描画分岐を次のようにする:
   - `isEndingScene` のとき: 選択肢グリッドと同じ場所・同じ Card スタイルで、横幅いっぱいの 1 ボタンを出す。ラベルは「FIN」を強調しつつ「アチーブメントを確認」を併記（V1 の `endingSceneLabel="FIN"` / `endingSceneHint="アチーブメントを確認してね"` を踏襲。最終的な文言は調整可）。押下で `onOpenEnding()` を呼ぶ。
   - それ以外で `hasSceneChoices` のとき: 従来どおり選択肢を描画。
   - どちらでもないとき: 何も出さない。
4. `HomePageV2` から `SceneOverlay` に `isEndingScene={isEndingScene}` と `onOpenEnding={() => setIsEndingDialogOpen(true)}` を渡す。

### 注意点
- エンディングシーンは選択肢ゼロなので、選択肢描画と FIN ボタンが二重に出ることはない（`isEndingScene` を優先分岐にすれば安全）。
- FIN ボタンの見た目は既存の選択肢 Card（`bgcolor: primary.main` / `borderRadius: "14px"`）に合わせ、画面内で浮かないようにする。
- アチーブメント判定ロジック（`achievements` 配列）と `EndingAchievementsDialog` はそのまま流用。BGM の `ending` 切り替えも現状維持。
