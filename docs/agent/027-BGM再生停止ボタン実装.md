# BGM 再生停止ボタン実装

## タスク概要

ゲーム画面に BGM の再生/停止トグルボタンを追加する。

### ユーザーフィードバック

「BGM 追加希望 (流れる流れないの2通りじゃなくて、ユーザがポチッとしたら流れる、もう一回ポチッとしたら止まるみたいな)」

### 前提調査

- `./docs/agent/026-BGM追加フィージビリティレポート.md` にて技術調査済み・LGTM
- 推奨実装: `HTMLAudioElement` + `useBgmPlayer` カスタムフック
- 音声ファイル: `webapp/frontend-react/public/bgm/banana-theme-64.mp3` (配置済み)
- 補足: 026 では仮の音源名として `/bgm/adventure-loop.mp3` を例示していたが、今回の実装では実ファイル名 `/bgm/banana-theme-64.mp3` に確定して進める

### 実装要件

- ボタンを押すと BGM が流れ始める
- もう一度ボタンを押すと BGM が止まる (トグル)
- ループ再生 (無限ループ)
- ユーザー操作起点で再生するため、Autoplay Policy に準拠

### 検討が必要な事項

- **ボタンの配置場所**: ゲーム画面 (`HomePage.tsx`) のどこに置くのが自然か
  - 現在の画面構成: 左側に シーン画像+テキスト+選択肢エリア、右側に バナナメーター+インベントリ
  - 参考: `webapp/frontend-react/src/pages/HomePage.tsx`
  - どのエリアに置くのが最もユーザー体験として自然か設計してほしい

## 詳細設計（プランナー記入欄）

### 1. ボタンの配置場所と理由

今回は、**左カラムのシーン表示 `Paper` 内の右上にオーバーレイ配置**でいきます。

- 配置先: `HomePage.tsx` のシーン画像を包んでいる `Paper`（`position: relative`）
- 位置: `top: 12px`, `right: 12px`, `zIndex: 3`（既存のテキストオーバーレイ `zIndex: 2` より前面）
- 理由1: BGM は「ゲーム全体設定」なので、シーン進行中にいつでも押せる位置が自然
- 理由2: 右カラム配置だと、スマホ表示時に下へ流れて見つけにくい。右上固定なら初見で見える
- 理由3: 選択肢カード群（画面下部）と操作目的が分離され、誤タップを起こしにくい

補足として、右カラム（バナナメーター付近）も候補だけど、今回は「いつでも即アクセス」を優先して却下します。

### 2. ボタンの UI デザイン方針

見た目は、ゲーム画面に馴染みつつ状態が一目で分かる形にします。

- コンポーネント: MUI `Button`（`variant="contained"`, `size="small"`）
- アイコン: `VolumeUpRounded`（再生中） / `VolumeOffRounded`（停止中）
- import 補足: アイコン利用のため `@mui/icons-material` から `VolumeUpRounded` / `VolumeOffRounded` を import する
- ラベル: `BGM ON` / `BGM OFF`
- 色:
  - 再生中: `success` 系（「流れてる」状態を直感的に伝える）
  - 停止中: `grey` 系（ニュートラル）
- アクセシビリティ:
  - `aria-label` に現在状態を含める（例: 「BGM を停止」/「BGM を再生」）
  - キーボード操作可能（標準 Button 挙動）

テキスト付き Button を採用するので、アイコンだけより迷いが少なく、ユーザーさんが「ポチッ」と押しやすいUIになります。

### 3. `useBgmPlayer` フックの実装内容

`026` の方針どおり `HTMLAudioElement` ベースで、`HomePage` から扱いやすい API を返します。

- 追加ファイル: `webapp/frontend-react/src/hooks/useBgmPlayer.ts`
- 固定音源: `const BGM_SRC = "/bgm/banana-theme-64.mp3"`
- 内部状態:
  - `isPlaying: boolean`
  - `errorMessage: string | null`
- 初期化 (`useEffect`):
  - `new Audio(BGM_SRC)` を生成
  - `loop = true`
  - `volume = 0.25`（初期値は控えめ）
- 公開メソッド:
  - `toggle`: `play()` / `pause()` の切り替え
  - `clearError`: `Snackbar` の `onClose` で呼び出し、エラーメッセージ表示だけを閉じる用途（`isPlaying` は変更しない）
- エラー処理:
  - `NotAllowedError` は「ブラウザ設定で再生がブロックされた」メッセージを設定
  - それ以外は汎用メッセージ + `console.error` で詳細ログ
- クリーンアップ:
  - アンマウント時に `pause()` して参照を破棄

### 4. `HomePage.tsx` の変更内容

実装変更は以下の4点に絞ります。

1. `useBgmPlayer` を import して呼び出し
2. シーン `Paper` の右上に BGM トグル `Button` を追加
3. ボタンに状態連動（`isPlaying` でアイコン・ラベル・色を切り替え）
4. エラー通知用 `Snackbar + Alert` を追加し、`errorMessage` を表示

実装イメージ（ざっくり）:

- `const { isPlaying, toggle, errorMessage, clearError } = useBgmPlayer();`
- `Paper` 内に `position: absolute` のボタン配置
- `Snackbar` は `Grid` の外に配置し、`return` 文の末尾で `Dialog` の後に置いてエラー通知

既存の `ResetButton`・ダイアログ・シーン選択ロジックには手を入れず、影響範囲を最小化します。

### 5. エラー時の UI フィードバック方針

エラーは「邪魔しすぎず、原因が分かる」出し方にします。

- 表示方式: `Snackbar`（`autoHideDuration: 4000`）+ `Alert`（`severity="warning"`）
- メッセージ例:
  - 自動再生制限系: 「再生がブロックされました。もう一度ボタンを押してください」
  - その他: 「BGM の再生に失敗しました。時間をおいて再試行してください」
- 再試行導線: ボタンはそのまま有効にしておき、ユーザーがすぐ再タップできる状態を維持

これで「失敗しても次の行動がわかる」体験にします。

### 1. ボタン配置の妥当性

**✅ LGTM です！**

### 2. MUI コンポーネント選択の妥当性

#### 🟡 **指摘 1: MUI アイコンの使い方**

設計ノートでは `VolumeUpRounded` / `VolumeOffRounded` を使う方針になってるけど、このプロジェクトではまだ MUI アイコンを使ってる箇所がないみたい。

実装時には `@mui/icons-material` からの import が必要になるよ。既存コードが `import { Button, ... } from "@mui/material"` みたいにまとめてるから、アイコンの import も

```tsx
import { VolumeUpRounded, VolumeOffRounded } from "@mui/icons-material";
```

ってスタイルで追加する形になるね。

**推奨**: 設計ノートの「2. ボタンの UI デザイン方針」に、MUI アイコンを使うための準備（`@mui/icons-material` からの import が必要）を明記しておくといいよ。

対応内容: 「2. ボタンの UI デザイン方針」に import 補足を追記し、`@mui/icons-material` から `VolumeUpRounded` / `VolumeOffRounded` を読み込む前提を明記したよ。

#### ✅ **Button コンポーネント選択は LGTM**

既存コードでも `Button` + `variant="contained"` を使ってる (`HomePage.tsx`:429) から、整合性バッチリだね。
