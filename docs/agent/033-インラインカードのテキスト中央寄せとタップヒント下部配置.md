# 033 - インラインカードのテキスト中央寄せとタップヒント下部配置

## 要望

- インラインメッセージカード内で、テキストが上部に寄っていて見づらい
- テキストは中央に配置したい
- 「タップして次へ」は下部に寄せたい

## 詳細設計

### 変更のゴール

1. `inlineMessage.kind === "scene"` のインラインカードで、メッセージ本文を縦横ともに中央寄せにする
2. `タップして次へ` ヒントはカード下部に固定して、本文と役割を分離する
3. 既存のシーン遷移ロジック（`advanceScene`）とタップ可能領域の広さは維持する

### 背景と現状整理

- 現在のシーン用インラインカードは、`CardContent` 内で本文→ヒントの順に通常フローで積んでいる
- そのため本文がカード上寄りに見えやすく、カードの中央に視線が集まりづらい
- 要望どおりにするには、カード内部を「中央表示エリア（本文）」と「下部固定エリア（ヒント）」に分けるのが最小差分で安全

### レイアウト方針

- 対象は `webapp/frontend-react/src/pages/HomePage.tsx` の `inlineMessage?.kind === "scene"` ブロックのみ
- `CardActionArea` の `minHeight: { xs: 140, sm: 168 }` はそのまま使う
- `CardContent` は `display: "flex"` + `flexDirection: "column"` + `height: "100%"` にして、カード内を上下 2 エリアに分割する
- 本文エリアは `Box` を 1 段追加し、`flex: 1` + `display: "flex"` + `alignItems: "center"` + `justifyContent: "center"` で中央固定にする
- 本文 `Typography` には `textAlign: "center"` を入れて、長文でも中央寄せの見た目を崩さない
- ヒントエリアは本文エリアの外側（`CardContent` 直下）に置き、`mt: "auto"` で下部へ寄せる
- `shouldShowSceneTapHint` と `tapHintBlink` の制御はそのまま利用し、表示タイミングと点滅演出は変えない

### UI差分の具体化

1. 変更対象コンポーネント
- `inlineMessage?.kind === "scene"` のカード描画ブロックだけを変更する
- `inlineMessage?.kind === "item"` や選択肢ボタン群の DOM 構造は触らない

2. 変更後の構造イメージ
```tsx
<CardActionArea onClick={...} sx={{ minHeight: { xs: 140, sm: 168 }, height: "100%" }}>
  <CardContent sx={{ height: "100%", display: "flex", flexDirection: "column", gap: 1 }}>
    <Box sx={{ flex: 1, display: "flex", alignItems: "center", justifyContent: "center" }}>
      <Typography sx={{ textAlign: "center", width: "100%" }}>{inlineMessage.text}</Typography>
    </Box>
    {shouldShowSceneTapHint && (
      <Box sx={{ mt: "auto", textAlign: "center", animation: `${tapHintBlink} 1.2s ease-in-out infinite` }}>
        タップして次へ
      </Box>
    )}
  </CardContent>
</CardActionArea>
```

3. 実装時の注意点
- `CardActionArea` の `onClick` は既存の `advanceScene` 呼び出しを維持し、クリック領域を狭めない
- ヒントを `position: "absolute"` にはしない（レスポンシブでの重なり事故を避けるため）
- `Typography` に `width: "100%"` を与えて中央揃えの基準を安定させる
- `gap` は最小値（`1`）にして本文とヒントの距離感だけを軽く確保する

### 実装ステップ

1. シーン用カードの JSX 構造を「本文ラッパー」と「ヒント表示」に分割
2. `CardContent` と本文ラッパーに `flex` 系 `sx` を追加して中央寄せと下部固定を成立させる
3. 既存条件分岐（`shouldShowSceneTapHint`）とアニメーション（`tapHintBlink`）がそのまま機能するか確認
4. 画面幅 `xs` / `sm` で本文の位置が崩れないことを目視確認
5. シーン進行、選択肢表示、アイテムメッセージ表示へ副作用がないことを手動確認

### 受け入れ条件

- シーン用インラインカード表示時、本文がカード中央に表示される
- `タップして次へ` が本文直下ではなく、カード下部に安定表示される
- カード全体タップで次シーンへ進む既存挙動は変わらない
- 選択肢表示、アイテム使用メッセージ、エンディング表示に副作用がない
- スマホ幅（`xs`）とデスクトップ幅（`sm` 以上）の両方で、本文とヒントが重ならない
