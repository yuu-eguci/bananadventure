# 033 - インラインカードのテキスト中央寄せとタップヒント下部配置

## 要望

- インラインメッセージカード内で、テキストが上部に寄っていて見づらい
- テキストは中央に配置したい
- 「タップして次へ」は下部に寄せたい

## 詳細設計

### 変更のゴール

1. `inlineMessage.kind === "scene"` のインラインカードで、メッセージ本文を縦横ともに中央寄せにする
2. `タップして次へ` ヒントはカード下部に固定して、本文と役割を分離する
3. 既存のシーン遷移ロジック（`advanceScene`）とタップ可能領域の広さは維持する

### 背景と現状整理

- 現在のシーン用インラインカードは、`CardContent` 内で本文→ヒントの順に通常フローで積んでいる
- そのため本文がカード上寄りに見えやすく、カードの中央に視線が集まりづらい
- 要望どおりにするには、カード内部を「中央表示エリア（本文）」と「下部固定エリア（ヒント）」に分けるのが最小差分で安全

### レイアウト方針

- 対象は `webapp/frontend-react/src/pages/HomePage.tsx` の `inlineMessage?.kind === "scene"` ブロックのみ
- `CardActionArea` の `minHeight: { xs: 140, sm: 168 }` はそのまま使う
- `CardContent` を `display: "flex"` + `flexDirection: "column"` + `height: "100%"` に変更して縦レイアウトを制御する
- 本文用の `Box` を 1 段追加し、`flex: 1` + `display: "flex"` + `alignItems: "center"` + `justifyContent: "center"` で中央寄せする
- 本文 `Typography` は `textAlign: "center"` を付けて、複数行でも中央配置の印象を崩さない
- タップヒントは本文エリアの外に置き、カード下端側に常に表示される構成にする（既存の点滅アニメーションは流用）

### 実装ステップ

1. シーン用インラインカードの `CardContent` に `height: "100%"` を追加し、縦方向の基準を固定
2. 本文 `Typography` を中央寄せ用 `Box` でラップして中央表示へ変更
3. `タップして次へ` の `Box` は本文ラップの外へ移動し、下部表示に固定
4. `onClick` の遷移処理・`shouldShowSceneTapHint` 判定・`tapHintBlink` は既存のまま維持

### 受け入れ条件

- シーン用インラインカード表示時、本文がカード中央に表示される
- `タップして次へ` が本文直下ではなく、カード下部に安定表示される
- カード全体タップで次シーンへ進む既存挙動は変わらない
- 選択肢表示、アイテム使用メッセージ、エンディング表示に副作用がない
