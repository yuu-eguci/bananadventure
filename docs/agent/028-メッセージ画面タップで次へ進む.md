# メッセージ画面タップで次へ進む

## タスク概要

選択肢を押したときに出るメッセージダイアログを、OK ボタン以外でも閉じて次へ進めるようにしたい。

### ユーザーフィードバック

「メッセージの OK が見えづらくて押しづらい」

### 今回の要望

- OK ボタンを押さなくても、画面のどこを押しても次へ進めるようにしたい
- 「どこを押しても大丈夫なメッセージ画面なんだ」とユーザーにわかるようにしたい

### 現状の実装

- `HomePage.tsx` の `Dialog` でメッセージを表示している
- `onCloseDialog` を `Dialog.onClose` と OK ボタンの両方に設定済み
- `DialogActions` 内に `Button OK` が配置されている

### 変更のゴール

1. **タップ/クリックで閉じる**: Dialog の外側 (Backdrop) をタップしても閉じる
2. **視覚的なヒント**: 「どこかタップして次へ」という旨をメッセージ画面内に表示する
3. OK ボタンは残すか廃止するかも含めて設計する

## 詳細設計（プランナー記入欄）

### 1. Dialog の「どこでもタップで閉じる」実装方針

今回は「Backdrop だけ」じゃなく、**メッセージ本体を含めてどこでもタップで進める**方針でいくよ。

- 前提として MUI v7 の `Dialog` には `disableBackdropClick` はないので使わない
- `Dialog` の `onClose` は残し、`reason` が `"backdropClick"` / `"escapeKeyDown"` のときに `onCloseDialog` を呼ぶ
- さらに `Dialog` 内側タップでも進めるため、`DialogContent`（または `slotProps.paper`）に `onClick` をつけて `onCloseDialog` を呼ぶ
- 連打で二重遷移しないように、`onCloseDialog` 側に「進行中フラグ」を入れて再入をブロックする

これで、ユーザー目線では「ダイアログが出ている間は、画面のどこを触っても次へ進む」体験になる。

### 2. ユーザーへの視覚的ヒント表示

「押せる場所が分からない」を防ぐため、ダイアログ下部にヒントを固定表示する。

- 表示文言: `タップして次へ`
- 表示位置: `DialogContentText` の下（`DialogContent` 内）
- 見た目: `Typography` の `caption` + `TouchApp` 系アイコンを横並び
- 動き: `opacity` を使ったゆるい点滅（例: 1.2s 無限ループ）で「次へ進める」ことを自然に伝える
- 補助: `DialogContent` 全体に `cursor: "pointer"` を付け、クリック可能な面だと分かるようにする

派手すぎる演出にはせず、読み終わったらすぐ次に進める軽い案内にする。

### 3. OK ボタンの扱い

今回は **OK ボタンは廃止** する。

- `DialogActions` ごと削除
- 理由1: 押下ターゲットを画面全体に広げる要望と完全一致する
- 理由2: 「OK を探して押す」認知コストをなくせる
- 理由3: モーダルの縦幅が少し減って、メッセージ本文とヒントが見やすくなる

閉じる導線は `Backdrop` / `Dialog 本体タップ` / `Esc` で確保する。

- `webapp/frontend-react/src/pages/HomePage.tsx`

  - ヒント UI（テキスト + アイコン + 点滅）追加
