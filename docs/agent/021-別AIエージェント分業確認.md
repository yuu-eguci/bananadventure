# 別 AI エージェント分業確認ノート

## タスク概要
- 目的: 【プランナー、実装者】に `codex`、【レビュワー】に `claude` を使い、別 AI エージェントで設計とレビューが回ることを再確認する
- 成果条件:
  - 設計ノート内に、プランナー本人 (自分が誰であるか) の記述がある
  - 設計ノート内に、レビュワー本人 (自分が誰であるか) の記述がある
  - 最後に統合者本人の記述を追記して完了する

## 今回の要望（原文要約）
- プランナー/実装者とレビュワーで、別 AI エージェントを使えることを確認したい
- 設計ノートの中に、プランナーとレビュワーがそれぞれ作業者自身のことを書き込む
- 最後に統合者が自分自身のことを書いて完了にする

## 詳細設計（プランナー記入欄）
- このノートを「分業の受け渡し場所」として使い、各ロールが同じファイル内で baton pass する
- 作業フローは次の 3 ステップで進める
  - Step 1: `codex` (プランナー、実装者) が詳細設計を記入し、自己記述セクションの「プランナー、実装者より」を埋める
  - Step 2: `claude` (レビュワー) がレビュー観点と指摘を「レビュー」へ記入し、自己記述セクションの「レビュワーより」を埋める
  - Step 3: 統合者 (Claude Code) が最終確認を行い、「統合者より」を記入して完了とする
- 確認観点:
  - 役割ごとに別 AI エージェント名が明示されていること
  - 各担当が「自分が誰か」を自己記述セクションに書いていること
  - 3 ロール分の記入が揃っていて、ノート単体で分業の完了状態が判断できること
- 完了条件:
  - 本ノート内で `codex`、`claude`、統合者の 3 者記述が揃っている
  - 設計、レビュー、統合の順番が本文から追える
  - Git 履歴上でもこの更新が確認できる
