# 035 - エンディング BGM 切り替え

## 要望

- 曲ファイルと曲名を定数で管理する
    - `banana-theme-64.mp3`: ばななちゃんのテーマ
    - `banana-theme-piano-64.mp3`: ばななちゃんのテーマ - ピアノ
- メイン画像エリアの外側の右上に、曲名を表示する
    - フォーマット: `♪ ばななちゃんのテーマ`
- エンディングのシーン (通常エンディングとゲームオーバーエンディング両方) では「ばななちゃんのテーマ」を再生
- それ以外のシーンでは「ばななちゃんのテーマ - ピアノ」を再生

## 現状

- メイン画像エリアの (重ねて) 右上に BGM OFF/ON ボタンがある
- `webapp/frontend-react/public/bgm/banana-theme-64.mp3` を再生している

## 詳細設計

### 調査で分かったこと

- BGM 再生は `useBgmPlayer.ts` カスタムフックが担当してる
    - `BGM_SRC` 定数 1 つで曲を固定、 `Audio` オブジェクトを 1 個だけ作って loop 再生してる
    - hook は `{ isPlaying, toggle, errorMessage, clearError }` を返す
- `HomePage.tsx` がこの hook を使って BGM ボタンを描画してる
    - ボタンはメイン画像 `Paper` の中に `position: absolute` で右上配置
- エンディング判定は `HomePage.tsx` 内の `ENDING_SCENE_IDS` (TRUE: 14, BAD: 15) で `isEndingScene` フラグを計算済み
- BGM ファイルは `public/bgm/` に 3 つある: `banana-theme-64.mp3`, `banana-theme-digital-64.mp3`, `banana-theme-piano-64.mp3`

### 変更方針

大きく 2 つのファイルを変更するよ。

#### 1. `useBgmPlayer.ts` - 曲切り替え機能の追加

今は 1 曲固定だけど、シーンに応じて曲を切り替えられるようにする。

- 曲の定数定義を追加する
    - `BGM_TRACKS` オブジェクトで曲ファイルと曲名をまとめて管理する
    - キーは `"main"` と `"ending"` の 2 つ
    - 各エントリに `src` (ファイルパス) と `label` (表示用の曲名) を持たせる

```ts
const BGM_TRACKS = {
  main: {
    src: "/bgm/banana-theme-piano-64.mp3",
    label: "ばななちゃんのテーマ - ピアノ",
  },
  ending: {
    src: "/bgm/banana-theme-64.mp3",
    label: "ばななちゃんのテーマ",
  },
} as const;

type BgmTrackKey = keyof typeof BGM_TRACKS;
```

- hook の引数に `trackKey: BgmTrackKey` を追加する
    - 呼び出し元 (HomePage) が現在のシーンに応じてどちらの曲を渡すか決める
- `trackKey` が変わったら曲を差し替える
    - `useEffect` で `trackKey` を監視する
    - `trackKey` が変わったとき:
        - 前の曲を pause する
        - 新しい `Audio` オブジェクトを作り直す (src を差し替え、 loop と volume を設定)
        - もし `isPlaying` 状態だったら新しい Audio を自動再生する (ユーザが BGM ON にしていた場合、曲が途切れないように)
- hook の返り値に `currentTrackLabel: string` を追加する
    - `BGM_TRACKS[trackKey].label` を返すだけ

#### 2. `HomePage.tsx` - 曲名表示と trackKey の受け渡し

- `useBgmPlayer` に渡す `trackKey` を決める
    - 既存の `isEndingScene` フラグを使う
    - `isEndingScene ? "ending" : "main"` でおしまい
- ただし `isEndingScene` は `viewModel` から計算するので、 `useBgmPlayer` の呼び出しより後に定義されてる
    - `trackKey` 計算を `useBgmPlayer` 呼び出しの前に移動するか、 `useMemo` などで整理する
    - 具体的には `currentSceneId` と `isEndingScene` の計算を hook 呼び出しの前に持ってくる
- BGM ボタンの外側 (メイン画像 `Paper` の外、右上寄せ) に曲名を表示する
    - フォーマット: `♪ ばななちゃんのテーマ`
    - `Typography` で `variant="caption"` くらい、控えめに右寄せで配置
    - BGM ボタンは今まで通り画像の上に重ねて配置 (移動しない)

### ファイル別の変更まとめ

| ファイル | やること |
|---|---|
| `useBgmPlayer.ts` | `BGM_TRACKS` 定数追加、 `trackKey` 引数追加、曲切り替え `useEffect` 追加、 `currentTrackLabel` を返り値に追加 |
| `HomePage.tsx` | `isEndingScene` 計算を前に移動、 `useBgmPlayer("ending" or "main")` に変更、曲名 `Typography` を `Paper` の外の右上に追加 |

## レビュー

### 指摘事項

1. `isPlaying` の stale closure リスク
    - `trackKey` 変更時の `useEffect` 内で「もし `isPlaying` だったら自動再生する」とあるけど、 `isPlaying` は state なので `useEffect` の依存配列に入れると曲切り替え以外でも effect が発火しちゃう
    - かといって依存配列に入れないと stale closure で古い値を参照するリスクがある
    - 対策: `isPlaying` を `useRef` で別途追跡するか、 `isPlayingRef` みたいな ref を state と並行して持つ方針を設計に明記してほしい

2. 初期状態の `trackKey` デフォルト値
    - `viewModel` が `null` の初期状態 (ローディング中) では `currentSceneId` が `-1` になって `isEndingScene` は `false` → `trackKey` は `"main"` になる
    - これは意図通りだと思うけど、設計に「初期状態は `"main"` がデフォルト」と一言あると実装者が迷わないよ

3. `BgmTrackKey` 型の export
    - `BgmTrackKey` を hook 内部だけで使うのか、 HomePage からも参照するのか明記されてない
    - HomePage 側で `trackKey` の変数に型注釈つけるなら export が必要になるので、設計に書いておいたほうがいいね

4. 既存の `BGM_SRC` 定数の削除
    - `BGM_TRACKS` に置き換わるから `BGM_SRC` は消すはずだけど、明示的に「削除する」と書いてないので一応

5. 曲名表示の条件
    - BGM が OFF のときも曲名は表示する？ それとも再生中だけ？ 要望には「表示する」としか書いてないので常時表示と読み取れるけど、確認しておいたほうがいいかも

6. Audio の `play()` 失敗ハンドリング
    - 曲切り替え時の自動再生で `play()` が失敗した場合 (ブラウザの autoplay policy など) のエラーハンドリングが設計に含まれてない
    - 既存の `toggle` には `NotAllowedError` のハンドリングがあるので、曲切り替え時にも同様の考慮が必要

### 軽微な確認事項

- `banana-theme-digital-64.mp3` が `public/bgm/` に存在するけど、要望でも設計でも使われてない。未使用ファイルとして残す認識で OK？
- 曲名表示の配置について「Paper の外の右上」とあるけど、具体的にどの親要素の中にどういう構造で配置するか (Box で wrap する等) もう少し具体的だと実装者に親切かも

### 総評

全体的な設計方針は妥当で、要望をちゃんとカバーしてる。大きな矛盾や漏れはないけど、上記の 1 と 6 は実装時にバグになりやすいところなので、設計段階で方針を決めておいてほしい。対応してもらったら LGTM 出すよ！
