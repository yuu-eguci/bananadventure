## 詳細設計
### 分析前提
- 対象データは `webapp/frontend-react/src/data/bananadventure-scenes.json` の現行版です。
- アイテム網羅の対象は `itemsOnSelect` から取得できる全アイテムです。
- 遷移判定は `SceneService` 実装どおり、`bananaMeter <= 0` のゲームオーバー遷移と `triggerItems` を含めて計算します。

### 分析結果
- 対象アイテムは 3 種です。  
  - `item 0`: 豆乳バナナ
  - `item 1`: ストロングゼロ
  - `item 2`: 六角レンチ・エンブレム・ハーブ
- 全探索の結果、**全アイテム網羅に必要な最小ルート数は 1 本**です。

### 全アイテム網羅ルート (1 本)
- 取得対象: `item 0, item 1, item 2`
- 終点: `scene 15`
- 開始前に `Reset` を押して、初期状態から開始します。

1. `scene 0` : `はじめる！`
2. `scene 1` : `コンビニに駆け込む` (item 0: 豆乳バナナを取得)
3. `scene 2` : `郵便物がないかチェックする`
4. `scene 3` : `行く！絶対行くのだ！`
5. `scene 4` : `ナビでしっかり経路を調べる`
6. `scene 5` : `うーちゃんと一緒に行くのだ`
7. `scene 6` : `隣の人に助けを求める` (item 1: ストロングゼロを取得)
8. `scene 7` : `一般客船「トンブラコ」`
9. `scene 8` : `備え付けの救護リュック` (item 2: 六角レンチ・エンブレム・ハーブを取得)
10. `scene 9` : `ビーチを散策`
11. `scene 11` : `食べるのだ！` (bananaMeter が 0 以下になり `scene 15` へ遷移)

### 補足
- このルートでは 11 手で全 3 アイテム取得を確認できます。
- `item 0` を持っている場合、`scene 10` 到達時は trigger により `scene 11` へ遷移しやすく、最終的に `scene 15` 終端になります。

### 検証観点
- 手順 11 完了時点でインベントリに `豆乳バナナ` `ストロングゼロ` `六角レンチ・エンブレム・ハーブ` の 3 件が表示されること。
- 途中でゲームオーバーにならず、最後に `scene 15` へ遷移すること。

### メンテ方針
- `webapp/frontend-react/src/data/bananadventure-scenes.json` の選択肢・delta・itemsOnSelect・triggerItems を変更した場合は、このルート表を再計算して更新します。
