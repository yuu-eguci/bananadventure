## 修正 1. メインセクションを 1:1 にしたら縦が大きすぎる → 全体を少し小さく

### 目的
sm 以上でメインセクションが `aspectRatio: 1/1` かつ幅 = 外枠 `maxWidth: 960` のため、最大 960×960 と縦に大きくなりすぎている。
正方形は維持したまま、**全体のサイズを一回り小さく**する。

### 原因
正方形の一辺 = メインセクションの幅 = 外枠コンテナの幅（`maxWidth: 960` 上限）。
よって幅の上限を下げれば、正方形の一辺（= 縦幅）も下がる。

### 実装手順
1. メインセクションの最大幅を表す定数を用意する（調整箇所を 1 つに集約する）。`MainSection.tsx` に置くと寸法系がまとまって分かりやすい。
   ```ts
   // メインセクション（および外枠）の最大幅。1:1 のため、この値が正方形の一辺の上限になる。
   export const MAIN_SECTION_MAX_WIDTH = 600; // 960 → 600（値は見ながら調整可）
   ```
2. `HomePageV2.tsx` の外枠 `Box` の `maxWidth: 960` を `MAIN_SECTION_MAX_WIDTH` に差し替える。
   - これで sm 以上の正方形が最大 600×600 程度になり、全体が一回り小さくなる。
   - xs（スマホ）は従来どおり固定高さ `MAIN_SECTION_HEIGHT.xs (520)` のままで影響しない。

### 注意点
- `maxWidth` を下げると、上部の `♪ {currentTrackLabel}` 表示や全体の横幅も一緒に縮む（= 「全体小さく」の意図どおり）。
- 値（600）は仮。実機で見ながら 560〜680 あたりで調整できる形にしておく。
- 正方形である以上、幅を縮めれば縦も同じだけ縮む。縦だけを縮めることはできない（仕様上 1:1 のため）点に留意。

## 修正 2. オーバーレイは「RATIO の位置 → メインセクション最下部」、見切れる時だけ下に伸ばす

### 目的・あるべき挙動
メッセージオーバーレイの縦範囲を、次の不変条件で厳密に定義する:

- 通常: オーバーレイの**上端 = `SCENE_OVERLAY_HEIGHT_RATIO` で決まる位置**、**下端 = メインセクション画像エリアの最下部**。
  （= 画像エリア下から `SCENE_OVERLAY_HEIGHT_RATIO` ぶんの帯）
- 例外: 中身がこの帯に収まらず**見切れるときだけ**、メインセクション最下部を超えて下方向へ伸ばす（その伸ばしぶんは延長ボックスで暗色を継ぐ）。

内容が少ないときに縦幅が大きく見えるのは、主に修正 1（正方形が大きい）に起因する。修正 1 で一辺が小さくなれば帯（= 一辺 × RATIO）も小さくなる。本修正では、その帯の不変条件を計算側で正しく保証し、リサイズしても崩れないようにする。

### 現状の計算（`useHomePageV2SceneOverlayLayout.ts`）
```ts
sceneOverlayTop = round(mainImageHeight * (1 - SCENE_OVERLAY_HEIGHT_RATIO))
overlayExtensionHeight = max(0, ceil(sceneOverlayTop + overlayScrollHeight - mainImageHeight))
```
この式自体は不変条件と整合している。実装で担保すべきは次の 2 点。

### 実装手順（確認・是正）
1. **`mainImageHeight` が常に「実レンダリングされた正方形の高さ」を指していることを保証する。**
   - `mainImageRef` は `MainSection` の画像エリア `Box`（`aspectRatio: 1/1` のもの）に付いていること。
   - 初期表示・ウィンドウリサイズ・ブレークポイント跨ぎで `ResizeObserver` が高さを更新していること。
   - もし初期値（`MAIN_SECTION_HEIGHT.xs = 520`）のまま sm 以上で更新されていないと、top が実寸とズレて帯が大きく見えるので、ここを重点確認する。
2. **「見切れた時だけ下に伸ばす」を担保する。**
   - 短い内容のとき: オーバーレイは `top → bottom:0` で帯の高さに収まり、`scrollHeight == clientHeight` のため `overlayExtensionHeight` は 0（= 伸びない）。
   - 長い内容のとき: 内容が帯を超えると `scrollHeight > clientHeight` となり、超過ぶんだけ `overlayExtensionHeight` が正になり下へ伸びる。
   - この挙動が保たれているか、長文シーン（例: scene 5 や、選択肢が多いシーン）と短文シーンの両方で確認する。
   - オーバーレイに `minHeight` 等の余計な下駄が入っていないこと（入っていると短文でも伸びてしまう）。

### 受け入れ条件
- 短い内容のシーン: オーバーレイは「RATIO 位置 → メインセクション最下部」の帯ぴったりに収まり、メインセクション下に余計な延長が出ない。
- 長い内容のシーン: 帯に収まらないぶんだけメインセクション下に延長され、内容が見切れない。延長部の背景はオーバーレイと同じ暗色。
- 修正 1 と合わせて、内容が少ないシーンでオーバーレイが過大に見えないこと。

### 注意点
- `SCENE_OVERLAY_HEIGHT_RATIO` は引き続き 1 箇所で調整できる定数のままにする。
- 修正 1 で正方形が小さくなると帯も比例して小さくなる。RATIO と `MAIN_SECTION_MAX_WIDTH` の両方を見ながら、内容量に対してちょうど良いバランスに調整する。

- 修正 2: 短文シーンで帯ぴったり・延長なし、長文シーンで見切れず下に伸びる。
