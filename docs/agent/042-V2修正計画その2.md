## 修正 2. オーバーレイの延長が消えない問題を直す

### あるべき挙動（ゴール）
- 通常時: オーバーレイは「`SCENE_OVERLAY_HEIGHT_RATIO` で決まる上端 → メインセクション画像の最下部」の帯にぴったり収まる。メインセクションの下に余計な延長を出さない。
- 例外時: 中身が帯に収まらないときだけ、収まらないぶんだけメインセクションの下へ伸びる。

### なぜ前回失敗したか（ルートコーズ。ここが一番大事）

前回は「内側 content の実寸を測れば直る」と考えて `SceneOverlay` に内側 `Box` を追加した。しかし **`ref={ref}` を外側 `Box` と内側 `Box` の両方に付けてしまった**（`SceneOverlay.tsx` の 30 行目と 44 行目の 2 箇所）。

React は ref を**子→親の順**で割り当てるため、最終的に `ref.current` は **外側 `Box`** を指す。つまり計測対象が内側 content ではなく、外側 `Box` になっていた。

外側 `Box` は `top` + `bottom: 0` で**引き伸ばされている**ので、その高さは「画像高さ + 延長」に依存する。これを延長量の計算に使うと、

```
外側Box高さ = (画像高さ + 延長) - top
延長        = top + 外側Box高さ - 画像高さ = 延長   ← 自分自身に戻る（フィードバックループ）
```

となり、**一度 80px の延長が出ると、その 80px が永遠に消えない**。中断記録の実測値がこれを裏付けている:

```
imageHeight: 600, overlayTopWithinImage: 240
計測された高さ = overlayHeight: 440  → 延長 = 240 + 440 - 600 = 80（消えない）
本来測るべき   = innerHeight: 358    → 延長 = 240 + 358 - 600 = -2 → 0（正解）
```

→ **直し方は「`ref` を内側 content `Box` だけに付ける」こと。** 内側 `Box` は通常フロー（引き伸ばされない）なので、その高さ＝中身の自然な高さになり、フィードバックループが消える。

### 触るファイルは 1 つだけ

- `webapp/frontend-react/src/components/HomePageV2/SceneOverlay.tsx`

> `useHomePageV2SceneOverlayLayout.ts` と `HomePageV2.tsx` は**変更不要**。
> （フックは `sceneOverlayContentRef` を `SceneOverlay` に渡しており、`SceneOverlay` 側が正しい要素にだけ ref を付ければ正しく動く。）

### やること（チェックリスト）

`SceneOverlay.tsx` の `return` の中で、`ref={ref}` が **2 箇所**にある。これを **内側の `Box` 1 箇所だけ**にする。

1. 外側 `Box`（`position: "absolute"`, `top`, `bottom: 0` を持つ方）の `ref={ref}` を **削除する**。
2. 内側 `Box`（`px: 2`, `py: 2`, `display: "flex"` を持つ方）の `ref={ref}` は **そのまま残す**。

それだけ。下に修正後の完成形を載せるので、`SceneOverlay.tsx` の `return (...)` 全体をこれに置き換えてよい。

### 修正後の `SceneOverlay.tsx`（このファイルでまるごと置き換えてOK）

```tsx
import { forwardRef } from "react";

import { Box, Card, CardActionArea, CardContent, Typography } from "@mui/material";

import { Scene, SceneChoice } from "@/models";

type Props = {
  scene: Scene | null;
  leadResponseText: string | null;
  top: number;
  isLoading: boolean;
  isEndingScene: boolean;
  onOpenEnding: () => void;
  onSelectChoice: (choice: SceneChoice) => void;
};

const ENDING_SCENE_LABEL = "FIN";
const ENDING_SCENE_HINT = "アチーブメントを確認";

const SceneOverlay = forwardRef<HTMLDivElement, Props>(function SceneOverlay(
  { scene, leadResponseText, top, isLoading, isEndingScene, onOpenEnding, onSelectChoice }: Props,
  ref,
) {
  const sceneChoices = scene?.sceneChoices ?? [];
  const hasSceneChoices = sceneChoices.length > 0;
  const shouldShowActionArea = isEndingScene || hasSceneChoices;

  return (
    // 外側 Box: 帯の暗い背景を担当。top → 画像下端（bottom:0）まで引き伸ばす。
    // ★ここには ref を付けない（引き伸ばされた高さを測ると延長計算がループするため）。
    <Box
      sx={{
        position: "absolute",
        left: 0,
        right: 0,
        top,
        bottom: 0,
        zIndex: 2,
        bgcolor: "rgba(0, 0, 0, 0.62)",
        color: "white",
        boxSizing: "border-box",
      }}
    >
      {/* 内側 Box: 中身（メッセージ＋選択肢）。通常フローなので高さ＝中身の自然な高さ。 */}
      {/* ★延長量の計算はこの要素の高さで行うので、ref はここだけに付ける。 */}
      <Box
        ref={ref}
        sx={{
          px: 2,
          py: 2,
          display: "flex",
          flexDirection: "column",
        }}
      >
        {leadResponseText ? (
          <>
            <Typography variant="subtitle1" sx={{ mb: 1 }}>
              {leadResponseText}
            </Typography>
            <Typography variant="subtitle1" sx={{ mb: 1, textAlign: "center", opacity: 0.7 }}>
              ***
            </Typography>
          </>
        ) : null}

        <Typography variant="subtitle1" sx={{ mb: shouldShowActionArea ? 1.5 : 0 }}>
          {scene?.text ?? ""}
        </Typography>

        {isEndingScene ? (
          <Box
            sx={{
              display: "grid",
              width: "100%",
              pb: 2,
            }}
          >
            <Card
              sx={{
                height: "100%",
                bgcolor: "primary.main",
                borderRadius: "14px",
                opacity: isLoading ? 0.7 : 1,
              }}
            >
              <CardActionArea sx={{ height: "100%" }} disabled={isLoading} onClick={onOpenEnding}>
                <CardContent sx={{ p: 2, textAlign: "center" }}>
                  <Typography variant="subtitle2" sx={{ fontWeight: 800, letterSpacing: "0.12em" }}>
                    {ENDING_SCENE_LABEL}
                  </Typography>
                  <Typography variant="body2">{ENDING_SCENE_HINT}</Typography>
                </CardContent>
              </CardActionArea>
            </Card>
          </Box>
        ) : hasSceneChoices ? (
          <>
            <Typography variant="subtitle2" sx={{ mb: 1 }}>
              選択肢
            </Typography>

            <Box
              sx={{
                display: "grid",
                gridTemplateColumns: { xs: "1fr", sm: "repeat(2, minmax(0, 1fr))" },
                gap: 1,
                width: "100%",
                pb: 2,
              }}
            >
              {sceneChoices.map((choice) => (
                <Card
                  key={choice.id}
                  sx={{
                    height: "100%",
                    bgcolor: "primary.main",
                    borderRadius: "14px",
                    opacity: isLoading ? 0.7 : 1,
                  }}
                >
                  <CardActionArea
                    sx={{ height: "100%" }}
                    disabled={isLoading}
                    onClick={() => {
                      onSelectChoice(choice);
                    }}
                  >
                    <CardContent sx={{ p: 2 }}>
                      <Typography variant="body2">{choice.text}</Typography>
                    </CardContent>
                  </CardActionArea>
                </Card>
              ))}
            </Box>
          </>
        ) : null}
      </Box>
    </Box>
  );
});

export default SceneOverlay;
```

## 検証手順（この通り確認する）

1. ビルドを通す。
   ```
   docker compose exec react-service yarn build
   ```
2. ブラウザで `http://127.0.0.1:5001/v2` を開く。
3. **短文シーンで延長が出ないこと**を確認する。
   - 初期シーン（タイトル「はじめる？」）や、テキストの短いシーンで、メインセクションの**下に黒い帯が余分にはみ出していない**こと。
   - 開発者ツールで、`MainSection` の `Paper` の高さが画像と同じ（sm 以上で約 600px）で、その下に余分な延長 `Box` が無い（高さ 0）こと。
4. **長文シーンで見切れず下に伸びること**を確認する。
   - テキストや選択肢が多いシーン（例: scene 5、選択肢 3 つ＋長文のシーン）で、中身が全部見え、収まらないぶんだけ下に黒帯が伸びていること。
5. **行き来しても延長が残らないこと**を確認する（前回の不具合）。
   - 長文シーン → 短文シーンへ進んだとき、延長が **0 に戻る**こと（80px が残らない）。

### 期待値の目安（imageHeight = 600, RATIO = 0.6 の場合）
- `sceneOverlayTop = 600 × (1 - 0.6) = 240`
- 帯の高さ = `600 - 240 = 360`
- 内側 content が 360 以下 → `overlayExtensionHeight = 0`（延長なし）
- 内側 content が例えば 500 → `overlayExtensionHeight = 240 + 500 - 600 = 140`（140px だけ伸びる）

### 受け入れ条件
- 短文シーン: 延長 0。帯ぴったり。
- 長文シーン: 見切れず、必要なぶんだけ延長。
- シーンを行き来しても延長が正しく 0 に戻る。
- `yarn build` が通る。
