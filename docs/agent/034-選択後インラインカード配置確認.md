# 選択後インラインカード配置確認

## タスク概要
- 目的: 選択肢タップ後に表示される「タップして次へ」付きインラインカードについて、本文中央配置とヒント下部配置の要望を満たしているか確認し、必要なら修正する
- 背景: ユーザ feedback では「本文が上揃え」「タップして次へが下揃えになっていない」と認識されている

## 今回の要望（原文要約）
- 選択肢タップ後に出るカードで、メッセージは中央に置きたい
- 「タップして次へ」はカード内部の下揃えにしたい

## 詳細設計（プランナー記入欄）
- 対象 UI は、シーン画面で選択肢をタップした直後に表示される、選択結果メッセージ入りのインラインカード
- 確認対象の実装箇所は `webapp/frontend-react/src/pages/HomePage.tsx`
- 選択後カードの表示起点は `onPressSceneChoice` で、`responseText` がある場合に `inlineMessage` を `kind: "scene"` としてセットしている
- カード本体は `inlineMessage?.kind === "scene"` の分岐内で描画されている
- 要望との対応関係は次の通り
  - 「メッセージは中央に置きたい」については、本文を包む `Box` に `flex: 1` と `alignItems: "center"` / `justifyContent: "center"` が入っており、カード内の残り高さに対して中央寄せになる構成
  - 「タップして次へはカード内部の下揃えにしたい」については、ヒント行に `mt: "auto"` が入っており、縦並びレイアウトの末尾へ押し下げる構成
- そのため、現状実装の時点で要望は満たしている前提で進める
- 修正方針は、まず現状レイアウトが実機表示でも崩れていないかを確認する。もし見え方に差がある場合は、`CardActionArea` / `CardContent` の高さ計算、または親コンテナの余白設定を追加で見直す
