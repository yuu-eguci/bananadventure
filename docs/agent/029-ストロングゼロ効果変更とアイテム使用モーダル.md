# ストロングゼロ効果変更とアイテム使用モーダル

## タスク概要

ユーザからの以下の feedback に対応する。

### ユーザーフィードバック

1. ストロングゼロアイテムの効果をバナナメーター -9 にしてほしい
2. アイテムをタップしたときにメッセージモーダルを表示してほしい
   - 「(アイテム名) を使用した！」
   - 「(ここに効果)」

### 今回の要望

- ストロングゼロの `bananaMeterEffect` を -9 に変更する
- アイテムタップ時に、028 と同様のモーダルで使用結果メッセージを表示する

## 詳細設計（プランナー記入欄）

### 変更のゴール

1. ストロングゼロ使用時の効果を、プレイヤーのばななメーター `-9` に統一する
2. アイテムボタン押下時に、028 と同じ操作感（どこタップでも閉じる）のメッセージモーダルを表示する
3. 既存の「シーン選択用モーダル」と「アイテム使用モーダル」を衝突させない

### 現状整理（実装前提）

- `HomePage.tsx` の `Dialog` は、現状 `onCloseDialog` でシーン遷移処理まで実行している
- そのため、アイテム使用で同じ close ハンドラをそのまま使うと、意図しないシーン遷移が混ざるリスクがある
- `SceneService.useItem` は `Item.bananaMeterDelta` を参照してメーター加減算している
- 現在のストロングゼロ JSON は `bananaMeterDelta: 0` / `bananaMeterDeltaPercent: -9` なので、実装上は効果が 0 扱いになっている

### 1. ストロングゼロ効果値の変更方針

現行モデルでは `bananaMeterEffect` というフィールドは使っていないので、今回の「効果 -9」は `bananaMeterDelta = -9` として反映する。

- 変更対象: `webapp/frontend-react/src/data/bananadventure-scenes.json`
- 対象アイテム: `id: 1`（ストロングゼロ）
- 変更内容:
  - `bananaMeterDelta: 0` → `bananaMeterDelta: -9`
  - `bananaMeterDeltaPercent` は現状未使用のため、今回仕様では参照しない

これで `SceneService.useItem` の既存ロジックだけで、使用時に確実に `-9` が反映される。

### 2. アイテム使用モーダルの状態設計

シーン用とアイテム用で close 後の挙動が違うので、モーダル種別を明示的に管理する。

- 追加 state（`HomePage.tsx`）
  - `dialogType: "scene" | "item" | null`
  - 既存の `openDialog` / `dialogMessage` は流用
- 開くときのルール
  - シーン選択時: `dialogType = "scene"` で開く
  - アイテム使用時: `dialogType = "item"` で開く
- 閉じるときのルール
  - `scene`: 既存どおりシーン遷移を実行
  - `item`: 遷移は実行せず、モーダルを閉じるだけ

この分離で、028 の UX（どこでもタップで閉じる）を維持しながら、挙動の混線を防ぐ。

### 3. アイテム押下フロー詳細

`onPressItem` を「効果適用→モーダル表示」まで含むフローにする。

1. `viewModel` がなければ return
2. `service.useItem({ viewModel, itemId: item.id })` を実行して `setViewModel` 反映
3. モーダル表示文言を作る
   - 1 行目: `{item.text} を使用した！`
   - 2 行目: 効果文（例: `ばななメーター -9`）
4. `dialogType = "item"` をセットして `openDialog = true`

補足:
- 押下ボタンは既存どおり `item.used` で `disabled` のため、使用済みアイテムは再表示されない
- `SceneService.useItem` 側のガード（未所持/使用済み）と UI 側の `disabled` で二重防御する

### 4. モーダル UI の実装方針（028 準拠）

- 見た目と操作は 028 と同様
  - `Dialog.onClose` で close
  - `slotProps.paper.onClick` で本文タップでも close
  - ヒント文（例: `どこでもタップして次へ`）は既存実装をそのまま共用
- アイテムモーダルの本文は 2 行表示にする
  - 1 行目: 使用通知
  - 2 行目: 効果通知

### 5. 変更対象ファイル

- `webapp/frontend-react/src/data/bananadventure-scenes.json`
  - ストロングゼロの `bananaMeterDelta` を `-9` へ変更
- `webapp/frontend-react/src/pages/HomePage.tsx`
  - `dialogType` 追加
  - `onCloseDialog` をモーダル種別で分岐
  - `onPressItem` のメッセージ作成・モーダル表示追加
  - 既存 Dialog は 028 の操作感を維持
- `docs/agent/029-ストロングゼロ効果変更とアイテム使用モーダル.md`
  - 本詳細設計の追記

### 受け入れ条件

- ストロングゼロを使用すると、ばななメーターが `-9` される
- アイテムボタン押下時にモーダルが開き、以下 2 行が表示される
  - `(アイテム名) を使用した！`
  - `(効果文)`
- モーダルは 028 と同様に、どこをタップしても閉じられる
- アイテムモーダルを閉じてもシーン遷移は発生しない
- 既存のシーン選択モーダルの遷移挙動は壊れない

### 非対象（今回やらないこと）

- アイテム効果文の i18n 化（現状 UI 方針に合わせて日本語ハードコード維持）
- `bananaMeterDeltaPercent` を使った割合計算ロジック追加
- 新規の演出（SE、専用アニメーション、専用アイコン）追加
