# 選択後インラインカードの縦配置を実表示どおり修正

## タスク概要
- 目的: 選択肢タップ後に表示される「タップして次へ」付きインラインカードで、本文を中央、ヒントをカード下部に見える形で配置する
- 背景: コード上では中央寄せ/下寄せの意図があるが、実際の表示では本文が上寄り、ヒントも中央付近に残っている

## 今回の要望（原文要約）
- 選択肢タップ後のカードで、メッセージを中央に置きたい
- 「タップして次へ」はカード内部の下揃えにしたい
- 現状は実表示が要望どおりになっていないので、バグとして修正したい

## 詳細設計（プランナー記入欄）
- 対象 UI: `HomePage` のシーン本文エリア内で、選択肢タップ後にだけ表示されるインラインカード（メッセージ本文 + 「タップして次へ」ヒント）
- 実表示の問題: コード上は本文を縦中央、ヒントをカード下端寄せにした意図に見えるが、実際の表示では本文が上寄りになり、ヒントもカード下部まで落ちず中央付近に残っている
- 想定原因: 親子の高さの受け渡しが中途半端で、`CardActionArea` に付けた `minHeight` / `height: 100%` が `CardContent` 側の実レイアウト高さとして効き切っていない可能性が高い。その結果、本文側の `flex: 1` が「余白を押し広げる対象」を持てず、`mt: auto` も期待した下寄せになっていないと考える
- 修正方針: インラインカード内で「カード全体の高さ」を 1 つの縦 flex コンテナに明確に持たせる。本文ブロックとヒントブロックが同じ高さ基準を共有できるようにし、本文は中央寄せ、ヒントは末尾配置になる構造へ寄せる。必要に応じて `Card` / `CardActionArea` / `CardContent` のどこか 1 箇所に最小高さを集約し、内側では `justifyContent: "space-between"` か、本文領域を `flexGrow: 1` にして中央寄せする形へ整理する
- 確認方法: React 画面で選択肢をタップし、インラインカード表示時に 1) 本文がカードの見た目中央に来ること 2) 「タップして次へ」がカード下寄りに張り付くこと 3) 画面幅 `xs` / `sm` の両方で縦位置が崩れないことを目視確認する

## 実装メモ（プランナー記入欄）
- `HomePage` のインラインカード描画部分を確認し、縦方向の高さ基準がどの要素で切れているかを整理する
- `scene` 用インラインカードのレイアウトを、本文とヒントが同じ縦コンテナ基準で並ぶ構造に組み替える
- 最小高さの責務を 1 箇所に寄せ、本文は中央寄せ、ヒントは下寄せになるよう `flex` の指定を見直す
- `xs` / `sm` の高さ差分が残るので、ブレークポイントごとに中央寄せが崩れないか確認する
- 修正後に選択肢タップでインラインカードを開き、実表示がノートの意図どおりかを確認する
