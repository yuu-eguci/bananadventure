# 選択後インラインカードの縦配置を実表示どおり修正

## タスク概要
- 目的: 選択肢タップ後に表示される「タップして次へ」付きインラインカードで、本文を中央、ヒントをカード下部に見える形で配置する
- 背景: コード上では中央寄せ/下寄せの意図があるが、実際の表示では本文が上寄り、ヒントも中央付近に残っている

## 今回の要望（原文要約）
- 選択肢タップ後のカードで、メッセージを中央に置きたい
- 「タップして次へ」はカード内部の下揃えにしたい
- 現状は実表示が要望どおりになっていないので、バグとして修正したい

## 詳細設計（プランナー記入欄）
- 対象 UI: `HomePage` のシーン本文エリア内で、選択肢タップ後にだけ表示されるインラインカード（メッセージ本文 + 「タップして次へ」ヒント）
- 実表示の問題: 現状は本文がカード上側に寄り、ヒントも本文のすぐ下に残るため、カード下部に余白があっても「本文中央 + ヒント下寄せ」の見え方になっていない
- レイヤーごとの責務整理:
  - `Card`: 背景色、角丸、横幅だけを持つ外枠に留める。高さ責務は持たせない
  - `CardActionArea`: タップ領域そのものとして扱い、`minHeight: { xs: 140, sm: 168 }` をここに集約する。今回のカードの見た目高さと操作可能高さはこの要素を基準にする
  - `CardContent`: `CardActionArea` の内側で本文とヒントを縦に並べる内部レイアウト専用コンテナにする。親高さを `%` で受け取るのではなく、`flex: 1` で残り領域を受け持つ
- なぜ現状崩れるのか: 現状は `Card` に高さ指定がなく、`CardActionArea` は `minHeight` のみ、`CardContent` は `height: 100%` という構成になっている。`height: 100%` は親の確定高さがないと有効にならないため、`CardContent` は実質的に中身ぶんの高さで収まる。その結果、本文ラッパーの `flex: 1` が伸びる先の高さを持てず、ヒント側の `mt: auto` も押し下げ先の余白を持てないので、本文もヒントも上側に詰まる
- 採用するレイアウト方針: `flexGrow + center` を採る。具体的には `CardActionArea` を高さ基準にし、その直下の `CardContent` を `flex: 1`, `display: "flex"`, `flexDirection: "column"` にする。本文ラッパーに `flexGrow: 1`, `display: "flex"`, `alignItems: "center"`, `justifyContent: "center"` を持たせ、ヒントはその後ろに通常フローで置く
- `space-between` を採らない理由: `justifyContent: "space-between"` だと本文ブロックはコンテナ先頭、ヒントは末尾に張り付くだけで、本文を中央に保つ責務が曖昧になる。ヒントの行高や `py` が `xs` / `sm` で変わると本文位置もぶれやすいため、本文専用の可変領域を `flexGrow: 1` で明示する方が意図が固定される
- `xs` / `sm` での確認基準:
  - `xs` は 599px 以下、`sm` は 600px 以上として確認する
  - `CardActionArea` の実高さが `xs` で 140px 以上、`sm` で 168px 以上あり、その余剰高さが `CardContent` 内の本文領域に渡っていること
  - 本文テキストの中心線が、「`CardContent` の上パディング端」から「ヒント行の上端」までの領域の中点から ±4px 以内に収まること
  - ヒント行の下端が `CardContent` の下パディング位置に揃い、カード下端からの見た目距離が `xs` で約 20px、`sm` で約 24px 前後になっていること
  - 実機相当確認は少なくとも `xs` で 390px 幅前後、`sm` で 768px 幅前後の 2 条件で行い、どちらでも本文が上寄りに見えないこと

## レビュー（レビュワー記入欄）
- **想定原因の仮説が曖昧**: 「高さの受け渡しが中途半端」とあるが、実際は `CardActionArea` の `minHeight` 指定だけで、`Card` 自体には高さ制約が入っていない。`CardContent` は `height: 100%` を指定されているが親の実高さが確定しないため効かないはず。原因分析をもう一段詳しく書くべき
- **修正方針が分散**: 「必要に応じて Card / CardActionArea / CardContent のどこか 1 箇所に最小高さを集約」とあるが、3 つの候補が並列で書いてあって迷いそう。`CardActionArea` か `Card` か、どちらに高さを確実に持たせるかの判断基準を明示してほしい
- **flexGrow と justifyContent の二択が混在**: 本文の中央寄せを `flexGrow: 1` にして `center` を指定するのか、`justifyContent: space-between` で親側に寄せるのか、最終的にどっちを採るのかが曖昧。これだと実装者が悩む
- **確認方法が抽象的**: 「見た目中央」と書いてあるが、たとえばカード高さ 168px なら本文ブロックの垂直中心がカード中心から ±数 px に収まる、みたいな具体的な目安を示した方が実装検証がぶれない
- **xs / sm の高さ差分に対する方針が薄い**: `minHeight: { xs: 140, sm: 168 }` のように高さがブレークポイントで変わる設計なのに、どのサイズでも中央寄せを維持するために必要な flex 配置の調整ポイントが説明不足
- **用語の揺れ**: 「カード全体の高さ」「縦 flex コンテナ」など表現が混在していて、最終的に `Card` / `CardActionArea` / `CardContent` のどのレイヤーを指しているのか分かりにくい箇所がある。一貫した用語で書いてほしい

**結論**: 問題意識自体は合ってるけど、原因分析の精度と修正方針の具体性が足りない。このままだと実装時に迷うし、実装者が独自判断でコードを書き直すリスクが高い。もう一度実コードの構造 (Card > CardActionArea > CardContent) と flex の責務を整理し直して、「どの要素に高さを持たせるか」「本文とヒントのレイアウトをどう組むか」を明確に書き直してほしいｗ

## 実装メモ（プランナー記入欄）
- `HomePage` の `inlineMessage?.kind === "scene"` 分岐を対象にし、`Card > CardActionArea > CardContent` の 3 層で責務を固定する
- `Card` は見た目の外枠だけに留め、`minHeight` は `CardActionArea` にだけ残す。`CardContent` の `height: "100%"` は撤去し、代わりに `flex: 1` を付けて親の高さ配分を受ける
- `CardActionArea` は `display: "flex"` と `alignItems: "stretch"` を維持し、必要なら `width: "100%"` を明示して、単一子要素の `CardContent` がタップ領域いっぱいに伸びる状態を固定する
- `CardContent` は `display: "flex"`, `flexDirection: "column"` にし、本文ラッパーへ `flexGrow: 1` と中央寄せを持たせる。ヒント側は `mt: auto` に頼らず、本文ラッパーが余白を吸収した後ろに置く
- `gap` は本文とヒントの最小間隔としてだけ使い、縦位置の主制御は `flexGrow: 1` に寄せる。これで `xs` から `sm` に増える 28px の高さ差分は本文領域側へ自然に配分される
- 確認時は選択肢タップ後カードで、`xs` / `sm` の両方について本文中心線とヒント下端位置を見比べ、上記確認基準から外れないことを確認する

## レビュー（レビュワー追記）

- **LGTM**: 前回の指摘を全部拾って、レイヤーごとの責務と高さ配分の仕組みを明確に整理できてる
- **原因分析がクリア**: 「`CardContent` の `height: 100%` は親の確定高さがないと効かない → `flexGrow: 1` が伸びる先を持てない → 上詰まりになる」という流れが一発で分かるようになった
- **採用方針の根拠が具体的**: `flexGrow + center` を採る理由と、`space-between` を避ける理由が両方書いてあって、実装者が迷わずに済む
- **確認基準が定量的**: 本文中心線の ±4px 以内、ヒント下端の見た目距離、`xs` / `sm` での実高さ目安が全部揃ってるので、検証時に「中央っぽい」で済まさずに判定できる
- **実装メモの具体性が高い**: `CardActionArea` に `minHeight` を集約する、`CardContent` は `flex: 1` で受ける、本文ラッパーに `flexGrow: 1` と `center` を持たせる、ヒントは後ろに通常フローで置くっていう手順が揃ってて、コード書く側が迷わない
- **用語の一貫性**: `Card` / `CardActionArea` / `CardContent` の 3 層表記で統一されてて、どの要素の話をしてるのかが明快
