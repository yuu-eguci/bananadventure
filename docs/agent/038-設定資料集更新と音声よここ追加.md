### 3. ページ状態の設計
- Accordion は今の「各項目が勝手に開閉する」形ではなく、`expandedItemId` を持つ制御コンポーネントに変える
- 状態は最低限これを持つ
  - `expandedItemId: LoreItemId | false`
  - `isVoiceEnabled: boolean`
  - `playingItemId: LoreItemId | null`
- 動作ルールは以下で固定する
  - 閉じているQをタップ: そのQを開く
  - 開いている同じQをもう一度タップ: 閉じる。再生中なら音声も止める
  - 別Qをタップ: 今の再生を止めてから新しいQを開き、そのQの音声シーケンスへ切り替える
  - `音声 OFF` のときは開閉だけ行い、音声は再生しない
- 状態遷移は次で固定する

| 操作 | `expandedItemId` | `playingItemId` | 備考 |
|---|---|---|---|
| 初期表示 | `false` | `null` | どのQも閉じている |
| Q1 をタップ、音声 OFF | `"q1"` | `null` | 開くだけ |
| Q1 をタップ、音声 ON | `"q1"` | `"q1"` | `Q1 -> A1` 再生開始 |
| Q1 再生完了 | `"q1"` | `null` | A は開いたまま、再生だけ終わる |
| 開いている Q1 を再タップ | `false` | `null` | 閉じる。再生中でも停止優先 |
| Q1 再生中に Q2 をタップ | `"q2"` | `"q2"` または `null` | 先に Q1 停止。音声 ON なら直後に Q2 再生、OFF なら `null` |
| Q1 を開いたまま `音声 OFF` | `"q1"` | `null` | UI 状態は維持、音だけ止める |
| Q1 を開いたまま `音声 OFF -> ON` | `"q1"` | `null` | 自動再生しない。次のQタップまで待つ |

- 設計上、「A が開いているのに `playingItemId` が `null`」は正常状態として許容する
  - これは `音声 OFF で開いたとき`
  - `Q音声 -> A音声` の再生が終わったあと
  - 再生途中に `音声 OFF` へ切り替えたあと
- 逆に `playingItemId !== null` なのに `expandedItemId === false` は禁止状態とする。`stop()` を呼ぶ経路では必ず `playingItemId` を先に `null` へ戻す

### 6. 音声 ON/OFF トグル
- `/` の BGM ボタンと同じく、トグル式の1ボタンで状態を切り替える
- ただし LorePage には大きなメイン画像がないので、ページ右上固定ではなくヘッダ右側に置く。見た目だけ `/` と揃え、レイアウト都合で配置は LorePage 用に自然な位置へ寄せる
- ボタン仕様
  - ラベル: `音声 ON` / `音声 OFF`
  - アイコン: `VolumeUpRounded` / `VolumeOffRounded`
  - ON 時は `color="success"` 寄り、OFF 時はグレー寄りで HomePage の BGM ボタンと同じ操作感にする
  - `aria-label` も ON/OFF に応じて切り替える
- 初期値は `OFF` にする
  - 理由1: `/` の BGM もユーザー操作で明示的に再生開始する設計だから、LorePage も揃えたほうが自然
  - 理由2: ブラウザの自動再生制限を避けやすい
- `音声 OFF` に切り替えた瞬間、再生中の音声があれば即停止する。Accordion の開閉状態は維持する

### 7. 音声再生の責務分割
- 音声制御は `LorePage.tsx` に全部書き込まず、`useLoreVoicePlayer.ts` のような専用 hook に切り出す
- hook の責務は以下
  - 単一の `HTMLAudioElement` を保持する
  - Q音声→A音声の2本を順番に再生する
  - 再生停止、再生中アイテム管理、連打時のキャンセルを担う
  - ページアンマウント時に確実に停止する
- hook から返すものはこのくらいで十分

```ts
type LoreVoiceSequence = {
  itemId: LoreItemId;
  questionSrc: string;
  answerSrc: string;
};

const {
  isVoiceEnabled,
  setVoiceEnabled,
  playingItemId,
  playSequence,
  stop,
} = useLoreVoicePlayer();
```

- `playSequence` の引数は `LoreItem` 全体ではなく `LoreVoiceSequence` に絞る
  - 理由: hook は再生責務だけを持ち、翻訳キーやリンク情報までは知らなくていい
  - 呼び出し側では `item.voice` と `item.id` から都度 `LoreVoiceSequence` を組み立てる
- 想定シグネチャはこれ

```ts
const playSequence = useCallback(async (sequence: LoreVoiceSequence) => {
  // ...
}, []);
```

### 8. 音声再生フロー
- Qタップ時の処理順はこうする
  1. タップされたQが現在開いているQと同じなら `expandedItemId` を `false` にする
  2. 同じQを閉じるケースなら `stop()` を呼んで終了
  3. 別Qを開くケースなら、まず今の再生を `stop()` する
  4. `expandedItemId` を新しいQへ更新する
  5. `isVoiceEnabled === true` のときだけ `playSequence({ itemId: item.id, questionSrc: item.voice.questionSrc, answerSrc: item.voice.answerSrc })` を呼ぶ
- `playSequence(sequence)` の中では `sequence.questionSrc` を再生し、終了イベントを受けたら `sequence.answerSrc` を再生する
- A音声まで終わったら `playingItemId` を `null` に戻す
- 開いているQを変えずに `音声 ON` にしただけでは自動再生しない。再生開始のトリガーはあくまで「Qをタップしたとき」に限定する
  - これで意図しない自動再生を防げる
- つまり、すでにQ1が開いている状態で `音声 OFF -> ON` に切り替えても、その場では `Q1 -> A1` を鳴らさない
  - 既に開いているQを聞きたい場合も、ユーザーがもう一度Qをタップして明示的に再生を起こす

### 9. 連打時・別Q切替時の競合対策
- ここは今回いちばん大事なので、実装方針を明確に固定する
- `requestIdRef` のような単調増加の識別子を hook 内に持ち、再生要求ごとに採番する
- 新しいQが押されたり OFF になった時点で `requestIdRef` を進め、古い再生要求を無効化する
- `playSequence` と `playSingleClip` は毎回「自分の requestId がまだ最新か」を確認してから次へ進む
- 停止処理では必ず以下を行う
  - `audio.pause()`
  - `audio.currentTime = 0`
  - `playingItemId = null`
  - 旧 requestId を無効化
- これで防ぐこと
  - 同じQを連打して Q音声が二重に鳴る
  - Q1 の A音声再生中に Q2 を押したら Q1 と Q2 が同時に鳴る
  - Q3 を開いた直後に OFF にしたのに、遅れて A3 が鳴り始める
- 音声要素は毎回複数生成せず、hook 内の単一 `Audio` を使い回す方針にする。複数インスタンスを並行で持たないことで競合点を減らす
- 採番と無効化のタイミングは以下で固定する

```ts
const requestIdRef = useRef(0);

const invalidateCurrentRequest = () => {
  requestIdRef.current += 1;
  return requestIdRef.current;
};

const stop = useCallback(() => {
  invalidateCurrentRequest();
  const audio = audioRef.current;
  if (audio) {
    audio.pause();
    audio.currentTime = 0;
  }
  setPlayingItemId(null);
}, []);

const playSequence = useCallback(async (sequence: LoreVoiceSequence) => {
  const requestId = invalidateCurrentRequest();
  setPlayingItemId(sequence.itemId);

  const playSingleClip = async (src: string) => {
    if (requestId !== requestIdRef.current) return false;

    const audio = ensureAudio();
    audio.pause();
    audio.currentTime = 0;
    audio.src = src;
    await audio.play();
    await waitForEndedOrCancellation(audio, requestId);

    return requestId === requestIdRef.current;
  };

  const questionCompleted = await playSingleClip(sequence.questionSrc);
  if (!questionCompleted) return;

  const answerCompleted = await playSingleClip(sequence.answerSrc);
  if (!answerCompleted) return;

  if (requestId === requestIdRef.current) {
    setPlayingItemId(null);
  }
}, []);
```

- increment は `playSingleClip` ごとではなく、新しい再生要求の開始時と停止時にだけ行う
  - `playSequence` 開始時: 旧要求をまとめて無効化
  - `stop` 実行時: 進行中要求を無効化
  - `playSingleClip` 開始時: 追加で increment しない。そうしないと Q音声からA音声へ正常遷移できなくなる
