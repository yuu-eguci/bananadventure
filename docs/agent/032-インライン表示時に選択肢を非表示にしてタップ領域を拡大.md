# 032 - インライン表示時に選択肢を非表示にしてタップ領域を拡大

## 今回の要望

Feedback:「タップして次へ」が選択肢のところに出るけど、そのときは "選択肢" いらないよね。

で、"選択肢" が消えたぶん、タップ領域は大きくしたい。タップできる範囲は大きい方が UX いいと思うし。

## 背景・現状

- 030 の実装で、選択肢ボタンが並ぶエリアに `inlineMessage.kind === "scene"` のとき
  インラインメッセージカードが表示されるようになった
- ただし見出しの `選択肢` はそのまま残っていて、「いま選ぶ状態ではない」場面でも見えてしまう
- インラインメッセージのカードは表示されるが、縦方向の余白が小さく
  「タップして次へ」の操作領域をもっと広げたい

## 詳細設計

### 変更のゴール

1. `inlineMessage.kind === "scene"` のときは `選択肢` 見出しを非表示にする
2. 同状態のインラインメッセージカードを大型化し、タップ領域を拡大する
3. 既存の遷移ロジック (`advanceScene`) やジングル演出は変更しない

### UI 制御方針

- シーン下部の分岐を以下の 2 パターンに整理する
  - `inlineMessage.kind === "scene"`: インラインメッセージ専用カードのみ表示
  - それ以外: 既存どおり `選択肢` 見出し + 選択肢カード群を表示
- これで「タップして次へ」の表示中に `選択肢` 文言が混ざらない状態にする

### タップ領域拡大の方針

- インラインメッセージカードを `width: 100%` の単独カードとして表示する
- `CardActionArea` に `minHeight` を設定して、縦方向のタップ可能範囲を明示的に拡張する
  - `xs: 140px`, `sm: 168px`
- `CardContent` の縦余白 (`py`) も増やし、視覚的にも「押せる領域」が伝わる見た目にする

### 実装対象

- `webapp/frontend-react/src/pages/HomePage.tsx`
  - シーンメッセージエリアの JSX 分岐を整理
  - `選択肢` 見出しの表示条件を `inlineMessage.kind !== "scene"` 時のみに変更
  - インラインメッセージカードの `CardActionArea` に最小高さを追加

### 受け入れ条件

- 「タップして次へ」表示中は `選択肢` 見出しが出ない
- 「タップして次へ」表示中に、カード全体で押しやすい縦サイズが確保される
- インラインカードをタップすると既存どおり次シーンへ進む
- 通常時の選択肢表示・アイテム表示・エンディング表示の既存挙動を壊さない

## レビュー

### 観点

1. 要望どおり `選択肢` 表示の抑制ができているか
2. タップ領域の拡大が UI 構造として明確か
3. 既存ロジックへの副作用が最小か

### 指摘と対応

1. 指摘: `選択肢` 見出しがインライン表示時も見えてしまう
   - 対応: JSX を 2 分岐に整理し、インライン表示時は見出しごと非表示へ変更
2. 指摘: インラインカードのタップ領域が狭い
   - 対応: `CardActionArea` に `minHeight` を追加し、`CardContent` の縦余白も拡張

### 結論

LGTM。この方針で実装して問題なし。

## 実装後追記

- 詳細設計どおりで完了。追加要件は発生しなかった

## オーナー向け要約

- 「タップして次へ」が出ている間は、`選択肢` 見出しを表示しない設計にした
- インラインメッセージカードを単独表示にして、通常の選択肢 UI と明確に切り替わるようにした
- タップ領域は `CardActionArea` の最小高さを増やして、スマホでも押しやすいサイズに拡大した
- シーン遷移ロジックやジングル演出には手を入れず、見た目と操作性に絞った最小差分で対応した
