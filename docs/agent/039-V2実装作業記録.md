# 039 - V2実装作業記録

## 要望

- 現在のメイン画面 `HomePage` の機能をベースにして、 UI を一新した `HomePageV2` を作る
- 既存 `HomePage` は長くなっているので、今回はコンポーネントをきちんと分けていく
- ゆっくり、段階を踏んで進める
- `AGENT` 用に、この実装作業の内容をノート `039` に追記していく

## 今回の進め方

- まず `HomePageV2` の空の土台を用意する
- 次に `HomePage` から `HomePageV2` へちょこちょこ確認できる導線を置く
- そのあとで、画面の中身を少しずつコンポーネントに分けていく

## 作業記録

- `HomePageV2` は空のページコンポーネントとして作成済み
- `App.tsx` に `/v2` ルートを追加済み
- `HomePage` 右下の固定ボタン群に `V2` ボタンを追加済み
- `HomePage` の `設定資料集` の横から `HomePageV2` に遷移できる状態になっている

### 1. `HomePageV2` の土台作成

- 新規ファイル `webapp/frontend-react/src/pages/HomePageV2.tsx` を作成
- 中身はまだ空にして、まずはページだけ存在する状態にした
- 将来ここに UI を分割して足していく

### 2. ルーティング追加

- `webapp/frontend-react/src/App.tsx` に `/v2` を追加
- `HomePageV2` を import して、通常の `HomePage` と並べて参照できるようにした

### 3. `HomePage` からの遷移導線追加

- `webapp/frontend-react/src/components/ResetButton.tsx` に `V2` ボタンを追加
- 既存の `設定資料集` ボタンと `Reset` ボタンの間に並べた
- これで `HomePage` を見ながら `HomePageV2` をすぐ開ける

### 4. メインセクションの追加

- `webapp/frontend-react/src/components/HomePageV2/MainSection.tsx` を追加
- `imageSrc` を props で受け取り、その画像を表示するだけの最小コンポーネントにした
- `webapp/frontend-react/src/pages/HomePageV2.tsx` から `"/sample-image/sample.png"` を渡して表示するようにした

### 5. V1 の見た目に寄せる調整

- `MainSection` を `HomePage` のメイン画像セクションに合わせて `Paper` ベースにした
- `borderRadius: "18px"` と `height: { xs: 520, md: 560 }` を反映した
- `overflow: "hidden"` と `objectFit: "cover"` を入れて、V1 の画像表示と同じ収まりにした
- `HomePageV2` 側も `maxWidth: 960`, `mx: "auto"`, `px: { xs: 0, sm: 1 }` にして、V1 の外枠と揃えた

### 6. BGM ボタンの UI 追加

- `MainSection` の右上に、V1 と同じ位置の BGM ボタン UI を置いた
- まだ機能は持たせず、見た目だけを置いている
- `VolumeOffRounded` と `BGM OFF` の表示にして、V1 の非再生状態の見た目に寄せた

### 7. BGM 再生機能の移植

- `HomePageV2` で `useBgmPlayer` を使うようにした
- `MainSection` の BGM ボタンを、V1 と同じく再生 / 停止できるようにした
- `currentTrackLabel` を上部に表示して、今鳴っている曲名が分かるようにした
- `BGM` のエラー表示を `Snackbar` と `Alert` で出すようにした
- ボタン UI は `BgmToggleButton` に切り出して、今後の変更をしやすくした
- `useBgmPlayer` は `trackKey` 変更時の Audio 差し替え処理を整理して、再利用しやすい形に少し整えた

### 8. メインセクションの責務整理

- `MainSection` は背景画像と overlay の受け皿だけに寄せた
- BGM の曲名表示はページ側へ戻した
- BGM ボタンは `MainSection` の子要素として載せる形にして、今後いろいろな UI を上に追加しやすくした

### 9. バナナメーターウィジェットの追加

- `webapp/frontend-react/src/components/HomePageV2/BananaMeterWidget.tsx` を新規作成
- バナナアイコン（36px）と数値を縦並びで表示するコンパクトなウィジェット
- BGM ボタンの直下（`top: 58px, right: 12px`）に absolute 配置
- 背景色はメインカラー（`primary.main`）、`Paper elevation={2}` でタップ可能な UI と統一した影を付与
- タップすると MUI の `Dialog` で説明文を表示する（モーダルもメインカラーで統一）
- 説明文: "バナナメーターだよ。ばななちゃんの体力を意味する。ゼロになると居酒屋TaTeTeへ飲みに行ってしまう（ゲームオーバー）。"
- `HomePageV2` からは仮値 `value={0}` を渡している（ゲーム状態が繋がったら差し替え）

### 10. アイテムウィジェットの追加

- `webapp/frontend-react/src/components/HomePageV2/ItemWidget.tsx` を新規作成
- バナナメーターウィジェットと同サイズ（アイコン 36px、`Paper elevation={2}`、メインカラー）で統一
- `Item` と `onUse` コールバックを props で受け取る
- 使用済みアイテムは `opacity: 0.4` + `elevation: 0`、クリック無効
- テキストは表示しない。タップすると `Dialog` でアイテム画像・説明・「使う」「キャンセル」ボタンを表示
- BGMボタン・バナナメーター・アイテムを1つの右パネル `Box`（flex column、`gap: 1`、`position: absolute top:12 right:12`）にまとめて配置
  - 各ウィジェットから個別の `position: absolute` を除去し、コンテナ側で位置管理するようにした
  - これにより余白が `gap` で自然に制御され、バナナメーターとアイテムの間が詰まる問題も解消した
- `HomePageV2` には `STUB_ITEMS` 配列ではなく `stubItem1` / `stubItem2` / `stubItem3` として個別定数で定義し、JSX でも個別に明示した
  - アイテムはゲーム進行中に1つずつ入手して出す仕様のため、`map` によるリスト描画は不適切と判断
- スタブの画像パスは `resolveImageUrl()` でラップして Vite の asset URL に変換済み
  - `public/item-image/` は空で `src/assets/item-image/` に実ファイルがあるため、パス直打ちでは 404 → sample 画像フォールバックになっていた
- スタブのアイテムデータはシーンデータ（`bananadventure-scenes.json`）の実データに合わせた
  - 豆乳バナナ（id:0）、ストロングゼロ（id:1）、六角レンチ・エンブレム・ハーブ（id:2）
- V2 内のすべての `Dialog` に `PaperProps={{ sx: { borderRadius: "18px" } }}` を適用し、メインセクションの丸み（`18px`）と統一
  - 対象: `ItemWidget`、`BananaMeterWidget`

### 19. `useHomePageV2Game` の新設

- `webapp/frontend-react/src/hooks/useHomePageV2Game.ts` を追加
- `viewModel`、`isLoading`、`leadResponseText` をフック内で保持する形にした
- マウント時に `SceneService.fetchInitialViewModel()` を呼び、初期シーンを読み込むようにした
- 公開 API は `scene`、`player`、`isLoading`、`leadResponseText`、`selectChoice`、`useItem`、`reset` にした
- `selectChoice` は V2 方針に合わせて、選択後すぐ次シーンへ進めつつ、選択肢の `responseText` を次シーン先頭用の `leadResponseText` として保持する形にした
- `reset` では初期シーンの再取得と `leadResponseText` のクリアを行う
- フック内に簡単な多重実行ガードを置いて、シーン遷移やアイテム使用の連打で状態が競合しにくいようにした

### 20. メッセージ・選択肢オーバーレイの実データ接続

- `webapp/frontend-react/src/components/HomePageV2/SceneOverlay.tsx` を追加し、固定プレビュー用だった `SceneOverlayPreview.tsx` は削除した
- `SceneOverlay` は `scene`、`leadResponseText`、`isLoading`、`onSelectChoice` を props で受け取り、実際の `scene.text` と `scene.sceneChoices` を描画するようにした
- V2 方針に合わせて、選択肢を押したあとの `responseText` は専用カードではなく、次シーンの本文の前に表示する構造にした
- `HomePageV2` を `useHomePageV2Game` へ接続し、メイン画像も `scene.image`、バナナメーター値も `player.bananaMeter` から取る形に切り替えた
- ただし今回の作業単位では、アイテム欄はまだスタブのままにしている

### 21. ローディング jingle backdrop の移植

- `webapp/frontend-react/src/components/HomePageV2/JingleBackdrop.tsx` を追加し、V1 の loading GIF 表示を V2 用の小コンポーネントとして分離した
- `HomePageV2` では `useHomePageV2Game` の `isLoading` を受けて `JingleBackdrop` を表示するようにした
- `useHomePageV2Game` 側では、初期化時とシーン遷移時の loading 表示時間を V1 に近い形に調整した
  - 初期表示時は 1 秒待ってから初期シーン取得を開始し、その後さらに 1 秒だけ jingle を残す
  - シーン選択後と reset 後は、状態更新後に 1 秒だけ jingle を残す
- これで、サービス呼び出しが速くても loading GIF が一瞬で消えず、V1 に近い体感になるようにした

### 22. アイテム入手・使用の接続

- `HomePageV2` の右パネルをスタブの固定アイテム表示から `player.items` の実データ描画へ切り替えた
- `ItemWidget` の `onUse` を非同期化し、 `useHomePageV2Game.useItem()` と接続した
- `ItemWidget` のモーダル内で「使う」を押したあと、同じモーダル内で `○○ を使用した！` と `ばななメーター +/-N` を表示する result view を追加した
- result view は自動で閉じず、ユーザが `閉じる` を押して閉じる導線にした
- 使用済みアイテムは既存どおり `opacity: 0.4` と `elevation: 0` のまま残し、再使用できない状態を保っている

### 23. バナナメーター連動・ゲームオーバー・トリガー確認

- バナナメーター表示自体は前段で `player.bananaMeter` に接続済みなので、今回は `SceneService` の振る舞いをテストで固定した
- `webapp/frontend-react/src/services/SceneService.test.ts` を追加した
- 確認した内容:
    - シーン選択で `bananaMeter` が増減すること
    - シーン選択で `bananaMeter <= 0` になったとき scene 15 ( game over ) へ遷移すること
    - scene 10 到達時に `豆乳バナナ` を所持していると、自動で trigger して scene 11 へ進み、アイテムが `used: true` になること
    - アイテム使用で `bananaMeter <= 0` になったときも scene 15 へ遷移すること

### 24. エンディング演出 (アチーブメントモーダル)

- `webapp/frontend-react/src/components/HomePageV2/EndingAchievementsDialog.tsx` を追加した
- V1 と同じ判定材料で、 `トゥルーエンド`、 `バッドエンド`、 `全アイテムコンプ` の 3 つをモーダル内に表示するようにした
- `HomePageV2` では scene 14 / 15 到達を検知して、自動でアチーブメントモーダルを開くようにした
- モーダルは V2 の統一方針に合わせて、メインカラー背景 + 角丸 `18px` にした

### 25. リセット導線

- `HomePageV2` に `ResetButton` を配置して、固定位置の `Reset` 導線を追加した
- `ResetButton` は `page` prop を受け取れるようにして、V2 上では `V2` 自己リンクの代わりに `V1` へ戻るボタンを表示するようにした

### 26. A〜G 実装後の修正計画の作成

- A〜G を実装して一通り動くようになったが、不具合・調整点があるため、修正の計画を立てた
- 計画はノート `041-V2修正計画.md` に、各修正を実装レベル（目的／対象ファイル／実装手順／注意点）で分解して記載した
- 対象の修正:
    1. V2 画面から V1 遷移ボタンを削除（`ResetButton`）
    2. メッセージオーバーレイの top を「メインセクション下から N%」の定数で決める（`useHomePageV2SceneOverlayLayout`）。長文時に下へ延長する仕様はキープ
    3. 選択肢レスポンスのフォントサイズを本文と同じ（`subtitle1`）にする（`SceneOverlay`）
    4. 選択肢レスポンスと本文の間に `***` を入れる（`SceneOverlay`）
    5. 画像比率を 1:1 にする。sm 以上は正方形、xs は高さ重視・横見切れ。高さが動的になるためレイアウト計測を実測ベースへ（`MainSection` / レイアウトフック）
    6. エンディングは自動でモーダルを出さず、最後のシーンの選択肢位置に「FIN｜アチーブメントを確認」ボタンを出してタップでモーダルを開く（`HomePageV2` / `SceneOverlay`）
- 実装順は、レイアウト計測を共有する修正 5 → 修正 2 をまとめて行う方針とした
