### 29. ノート 042 の修正実装

- `MainSection` に `MAIN_SECTION_MAX_WIDTH` 定数を追加し、 `HomePageV2` の外枠 `maxWidth` へ適用した。これで sm 以上の 1:1 メインセクションが最大 `600 x 600` に収まるようにした
- `useHomePageV2SceneOverlayLayout` は、 overlay の外側ではなく内側 content の実寸を `ResizeObserver` で監視するようにした
- これにより、長文 scene のあとに短文 scene へ移ったときも `overlayExtensionHeight` が即座に再計算され、必要ない延長が残らないようにした
- overlay の基本ルールは維持している。通常は `SCENE_OVERLAY_HEIGHT_RATIO` で決まる帯に収まり、内容が見切れるときだけ下へ延長する
- その後、外側 overlay 自体を測る実装だと、一度下へ伸びたあと短文 scene に戻っても延長が残るケースを確認したため修正した
- `SceneOverlay` に内側の content wrapper を追加し、 `useHomePageV2SceneOverlayLayout` はその content 実寸だけを `ResizeObserver` で測る形に変更した
- これで `overlayExtensionHeight` は「本当に content が帯に収まらないぶん」だけになるようにした
- Browser で確認しながら `SCENE_OVERLAY_HEIGHT_RATIO` も `0.45 -> 0.6` に調整し、短文 scene が通常帯の中に収まりやすいバランスへ寄せた

### 31. 中断を踏まえた 042 の精密化（ルートコーズ特定）

- 30 の中断記録の数値からルートコーズを特定した
- 原因: `SceneOverlay.tsx` で forward した `ref` を**外側 Box と内側 Box の両方**に付けていた。React は ref を子→親の順で割り当てるため `ref.current` が外側 Box（`top` + `bottom:0` で引き伸ばされる要素）を指し、その高さで延長量を計算していた
- これにより `延長 = top + 外側Box高さ - 画像高さ = 延長` という自己参照のフィードバックループになり、一度出た `80px` の延長が消えなくなっていた（実測 `overlayHeight:440` で計算 → 80px。本来は `innerHeight:358` で計算 → 0）
- 直し方: **`ref` を内側 content Box だけに付ける**（外側 Box の `ref` を消す）。内側 Box は通常フローで中身の自然な高さになり、ループが消える
- フック（`useHomePageV2SceneOverlayLayout.ts`）と `HomePageV2.tsx` は変更不要であることも確認
- ノート `042-V2修正計画その2.md` を、能力の低い実装者でも完遂できるよう精密版に更新した
    - 修正 1（メインセクション縮小）は実装済みとして整理
    - 修正 2 はルートコーズ説明 + コピペ可能な `SceneOverlay.tsx` の完成形 + 期待値つきの検証手順を記載

### 32. ノート 042（精密版）の修正 2 実装

### 36. メッセージスピード機能の本実装（1文字ずつ描画 + 描画後に選択肢）（タスク 27）

- 設計
    - 新規フック `useTypewriter(fullText, charDelayMs, enabled)` を追加。`fullText` が変われば最初から打ち直し、`charDelayMs === 0`（光速）は一括表示、`enabled === false`（ローディング中で JingleBackdrop に隠れている間）は進めない（表示されてから打ち始める）。`skip()` で残りを一気に表示
    - `SceneOverlay` で `leadResponseText + scene.text` を 1 本のテキストとして typewriter にかけ、表示位置で lead / `***` / 本文にスライスして描画。`***` は lead 打ち終わってから表示
    - 選択肢／FIN は `isComplete`（描画完了）のときだけ表示。`Fade` でふわっと出す
    - 打鍵途中に内側 content をタップすると `skip()`（VN 定番。選択肢はまだ無いので誤タップなし）
    - overlay 延長は既存の ResizeObserver が文字増加・選択肢出現を拾って自動再計算するため、レイアウト側は変更不要
- 配線
    - `HomePageV2` から `MESSAGE_SPEEDS[messageSpeed].charDelayMs` を `SceneOverlay` の `charDelayMs` prop へ渡す
    - スピードは `useMessageSpeed`（localStorage 永続）から取得。システムモーダルで変更したスピードが即座に反映される
- `docker compose exec react-service yarn build` は通した
