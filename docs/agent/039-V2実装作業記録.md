# 039 - V2実装作業記録

## 要望

- 現在のメイン画面 `HomePage` の機能をベースにして、 UI を一新した `HomePageV2` を作る
- 既存 `HomePage` は長くなっているので、今回はコンポーネントをきちんと分けていく
- ゆっくり、段階を踏んで進める
- `AGENT` 用に、この実装作業の内容をノート `039` に追記していく

## 今回の進め方

- まず `HomePageV2` の空の土台を用意する
- 次に `HomePage` から `HomePageV2` へちょこちょこ確認できる導線を置く
- そのあとで、画面の中身を少しずつコンポーネントに分けていく

## 作業記録

- `HomePageV2` は空のページコンポーネントとして作成済み
- `App.tsx` に `/v2` ルートを追加済み
- `HomePage` 右下の固定ボタン群に `V2` ボタンを追加済み
- `HomePage` の `設定資料集` の横から `HomePageV2` に遷移できる状態になっている

### 1. `HomePageV2` の土台作成

- 新規ファイル `webapp/frontend-react/src/pages/HomePageV2.tsx` を作成
- 中身はまだ空にして、まずはページだけ存在する状態にした
- 将来ここに UI を分割して足していく

### 2. ルーティング追加

- `webapp/frontend-react/src/App.tsx` に `/v2` を追加
- `HomePageV2` を import して、通常の `HomePage` と並べて参照できるようにした

### 3. `HomePage` からの遷移導線追加

- `webapp/frontend-react/src/components/ResetButton.tsx` に `V2` ボタンを追加
- 既存の `設定資料集` ボタンと `Reset` ボタンの間に並べた
- これで `HomePage` を見ながら `HomePageV2` をすぐ開ける

### 4. メインセクションの追加

- `webapp/frontend-react/src/components/HomePageV2/MainSection.tsx` を追加
- `imageSrc` を props で受け取り、その画像を表示するだけの最小コンポーネントにした
- `webapp/frontend-react/src/pages/HomePageV2.tsx` から `"/sample-image/sample.png"` を渡して表示するようにした

### 5. V1 の見た目に寄せる調整

- `MainSection` を `HomePage` のメイン画像セクションに合わせて `Paper` ベースにした
- `borderRadius: "18px"` と `height: { xs: 520, md: 560 }` を反映した
- `overflow: "hidden"` と `objectFit: "cover"` を入れて、V1 の画像表示と同じ収まりにした
- `HomePageV2` 側も `maxWidth: 960`, `mx: "auto"`, `px: { xs: 0, sm: 1 }` にして、V1 の外枠と揃えた

### 6. BGM ボタンの UI 追加

- `MainSection` の右上に、V1 と同じ位置の BGM ボタン UI を置いた
- まだ機能は持たせず、見た目だけを置いている
- `VolumeOffRounded` と `BGM OFF` の表示にして、V1 の非再生状態の見た目に寄せた

### 7. BGM 再生機能の移植

- `HomePageV2` で `useBgmPlayer` を使うようにした
- `MainSection` の BGM ボタンを、V1 と同じく再生 / 停止できるようにした
- `currentTrackLabel` を上部に表示して、今鳴っている曲名が分かるようにした
- `BGM` のエラー表示を `Snackbar` と `Alert` で出すようにした
- ボタン UI は `BgmToggleButton` に切り出して、今後の変更をしやすくした
- `useBgmPlayer` は `trackKey` 変更時の Audio 差し替え処理を整理して、再利用しやすい形に少し整えた

### 8. メインセクションの責務整理

- `MainSection` は背景画像と overlay の受け皿だけに寄せた
- BGM の曲名表示はページ側へ戻した
- BGM ボタンは `MainSection` の子要素として載せる形にして、今後いろいろな UI を上に追加しやすくした

### 9. バナナメーターウィジェットの追加

- `webapp/frontend-react/src/components/HomePageV2/BananaMeterWidget.tsx` を新規作成
- バナナアイコン（36px）と数値を縦並びで表示するコンパクトなウィジェット
- BGM ボタンの直下（`top: 58px, right: 12px`）に absolute 配置
- 背景色はメインカラー（`primary.main`）、`Paper elevation={2}` でタップ可能な UI と統一した影を付与
- タップすると MUI の `Dialog` で説明文を表示する（モーダルもメインカラーで統一）
- 説明文: "バナナメーターだよ。ばななちゃんの体力を意味する。ゼロになると居酒屋TaTeTeへ飲みに行ってしまう（ゲームオーバー）。"
- `HomePageV2` からは仮値 `value={0}` を渡している（ゲーム状態が繋がったら差し替え）

### 10. アイテムウィジェットの追加

- `webapp/frontend-react/src/components/HomePageV2/ItemWidget.tsx` を新規作成
- バナナメーターウィジェットと同サイズ（アイコン 36px、`Paper elevation={2}`、メインカラー）で統一
- `Item` と `onUse` コールバックを props で受け取る
- 使用済みアイテムは `opacity: 0.4` + `elevation: 0`、クリック無効
- テキストは表示しない。タップすると `Dialog` でアイテム画像・説明・「使う」「キャンセル」ボタンを表示
- BGMボタン・バナナメーター・アイテムを1つの右パネル `Box`（flex column、`gap: 1`、`position: absolute top:12 right:12`）にまとめて配置
  - 各ウィジェットから個別の `position: absolute` を除去し、コンテナ側で位置管理するようにした
  - これにより余白が `gap` で自然に制御され、バナナメーターとアイテムの間が詰まる問題も解消した
- `HomePageV2` には `STUB_ITEMS` 配列ではなく `stubItem1` / `stubItem2` / `stubItem3` として個別定数で定義し、JSX でも個別に明示した
  - アイテムはゲーム進行中に1つずつ入手して出す仕様のため、`map` によるリスト描画は不適切と判断
- スタブの画像パスは `resolveImageUrl()` でラップして Vite の asset URL に変換済み
  - `public/item-image/` は空で `src/assets/item-image/` に実ファイルがあるため、パス直打ちでは 404 → sample 画像フォールバックになっていた
- スタブのアイテムデータはシーンデータ（`bananadventure-scenes.json`）の実データに合わせた
  - 豆乳バナナ（id:0）、ストロングゼロ（id:1）、六角レンチ・エンブレム・ハーブ（id:2）
- V2 内のすべての `Dialog` に `PaperProps={{ sx: { borderRadius: "18px" } }}` を適用し、メインセクションの丸み（`18px`）と統一
  - 対象: `ItemWidget`、`BananaMeterWidget`

### 11. メインセクション下側の UI 予備実装

- `webapp/frontend-react/src/components/HomePageV2/SceneOverlayPreview.tsx` を追加
- V1 のメッセージ表示と選択肢表示の見た目を、メインセクションの下側 `bottom` に重ねるプレビューとして置いた
- 右側ウィジェットと同じく、画像の上に載るレイヤーとして `MainSection` の子要素に追加した
- まだ機能は持たせず、レイアウトと見た目だけを確認できる状態にしてある
- `yarn build` で型崩れがないことを確認した

### 12. シーンメッセージ表示の調整

- V2 ではシーンメッセージは必要なままにして、`「タップして次へ」` のヒントだけ外す方針に切り替えた
- 下部のプレビューはメッセージ表示と選択肢表示を残したまま、タップヒントだけを消す想定に直した

### 13. 選択肢ボタンの縦位置調整

- V2 の選択肢ボタンを、 V1 に寄せて `CardActionArea` + `CardContent` の構造に戻した
- ボタン内の縦中央寄せをやめて、 V1 と同じ素直な収まりに戻した

### 14. メッセージオーバーレイと右上ウィジェットの重なり防止

- 右上の `BGM` ボタン、バナナメーター、アイテム群の高さを計測して、メッセージオーバーレイの `top` を動的に決めるようにした
- これで、 overlay の頂点がボタン群の下に必ず来るようにした

### 15. メッセージオーバーレイが長いときの下方向拡張

- `SceneOverlayPreview` の実高さを見て、メインセクションの下を必要なぶんだけ伸ばすようにした
- これで overlay がメインセクションの下で見切れる場合でも、画像の下側に余白を足して収められるようにした

### 16. 下側余白と延長部分の色合わせ

- 一番下の選択肢の下に余白を足した
- 伸ばしたぶんの背景色をメッセージオーバーレイと同じ暗色にそろえた

### 17. V2 レイアウトの整理

- `webapp/frontend-react/src/hooks/useHomePageV2SceneOverlayLayout.ts` を追加して、右上ウィジェット群の高さ測定と overlay の延長高さ計算を分離した
- `webapp/frontend-react/src/components/HomePageV2/HomePageV2RightPanel.tsx` を追加して、右上の共通レイアウトをまとめた
- `MainSection` の下側拡張は `overlayExtensionHeight` という意味のある名前に整理した
- `HomePageV2` から計測ロジックを外して、画面構成の見通しを少しよくした
