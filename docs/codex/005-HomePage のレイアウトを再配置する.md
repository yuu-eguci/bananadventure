HomePage のレイアウトを再配置する
===

## 背景
現状の HomePage は、左カラムにシーン画像とメッセージ、右カラムに選択肢ボタンとインベントリがある。  
UI の見た目は良いが、視線の流れとして「メッセージを見る → 選択肢を選ぶ」が分断されている。

## 今回の要望
- 右側カラムにある選択肢ボタンを、左側カラムのメッセージエリアの下へ移動する。
- 選択肢は 2x2 配置で見せる。
- 右側カラムは狭くして、左側カラムをより中央寄せに見せる。
- 今回はユーザ確認を優先するため、レビューは軽めにする。

## ゴール
- メッセージエリアの直下に選択肢カードが 2 列で並ぶ。
- 右カラムは従来より狭くなる。
- モバイル ( `xs` ) でも崩れず表示される。

## 非ゴール
- ゲームロジックや選択肢遷移処理の変更。
- カード色や文言などの大幅なビジュアル刷新。

## 変更対象の候補
- `webapp/frontend-react/src/pages/HomePage.tsx`
- `docs/codex/005-HomePage のレイアウトを再配置する.md`

## 進め方
このノートで先に詳細設計を固めて、 LGTM 後に実装へ進む。

## 詳細設計
### 1. レイアウト構成の変更
- 左カラムを `md:8`、右カラムを `md:4` に変更して、左カラム比重を上げる。
- メッセージ付き画像の `Paper` は左カラム上部に維持する。
- 選択肢セクションを右カラムから左カラムへ移動し、画像下に配置する。

### 2. 選択肢の 2x2 表示
- 選択肢一覧は `Grid container` で描画し、各カードを `xs:12, sm:6` で配置する。
- これで PC では 2 列、モバイルでは 1 列になる。
- 選択肢数が 4 件を超える場合も 2 列グリッドで自然に折り返す。

### 3. 右カラムの役割
- 右カラムには `バナナメーター` と `インベントリ` を残す。
- 「選択肢」見出しとカード群は右カラムから削除する。
- 右カラム幅を縮めることで、視覚的な主役を左カラムへ寄せる。

### 4. 影響範囲
- クリック処理 ( `onPressSceneChoice` ) やダイアログ挙動は変更しない。
- データ取得や Scene 遷移ロジックは変更しない。

### 5. 実装タスク ( 予定 )
1. `HomePage.tsx` の左右 `Grid size` を `7/5` から `8/4` に変更する。  
2. 選択肢見出しと選択肢カード描画ブロックを左カラムへ移設する。  
3. 選択肢カードを 2 列グリッド (`xs:12, sm:6`) で描画する。  
4. 右カラムから重複要素を削除し、メーターとインベントリだけにする。  

### 6. 確認観点
- デスクトップ幅で選択肢カードが 2 列で表示されること。
- モバイル幅で選択肢カードが 1 列表示になること。
- 選択肢クリックで従来どおりダイアログが開くこと。
- 右カラムが従来より狭くなっていること。

### 7. 受け入れ条件
- 「メッセージ → 選択肢」の順で視線移動しやすい配置になっている。  
- 選択肢が画像エリア下に 2 列レイアウトで配置される。  
- 右カラム縮小後もバナナメーターとインベントリ表示が崩れない。
