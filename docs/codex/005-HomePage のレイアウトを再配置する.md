HomePage のレイアウトを再配置する (統合版)
===

## 背景
HomePage のレイアウト改善で、次の 2 段階を実施した。

1. 第 1 段階
- 選択肢を右カラムから左カラム側へ移動。
- 視線導線を「メッセージ → 選択肢」に寄せる。

2. 第 2 段階 (再設計)
- 画像オーバーレイ内のメッセージ + 選択肢で、右ズレ / 欠けが発生。
- オーバーレイ実装を再設計して、幅計算を安定化。

## 今回の要望 (統合)
- 選択肢をメッセージ直下に置く。
- 右カラムを狭くして主役を左へ寄せる。
- オーバーレイのズレ / 欠けを解消する。
- 選択肢ボタン色は従来トーン ( `primary.main` ) を維持する。
- 選択肢が空のときは `選択肢` ラベルを非表示にする。

## ゴール
- メッセージ + 選択肢オーバーレイが画像幅に対して正しく収まり、左右にはみ出さない。
- 選択肢はメッセージ直下で 2 列 ( `sm` 以上 ) / 1 列 ( `xs` ) で表示される。
- ボタン色は旧トーンに一致する。
- 選択肢が 0 件なら `選択肢` ラベルは表示しない。

## 非ゴール
- シーン遷移ロジックやダイアログ仕様の変更。
- インベントリ機能の UI 刷新。

## 変更対象
- `webapp/frontend-react/src/pages/HomePage.tsx`
- `docs/codex/005-HomePage のレイアウトを再配置する.md`

## 詳細設計
### 1. レイアウト構成
- 左カラム `md:8`、右カラム `md:4` を採用。
- 右カラムには `バナナメーター` と `インベントリ` のみを配置。
- ページ全体は `maxWidth: 960` で中央寄せ。

### 2. オーバーレイ構成
- 画像下部オーバーレイを `position: absolute` で固定。
- 境界は `left: 0` / `right: 0` / `bottom: 0` を明示。
- `boxSizing: border-box` を適用し、padding 起因のはみ出しを防ぐ。

### 3. 選択肢レイアウト
- オーバーレイ内部は CSS Grid (`Box`) を使う。
- 列定義: `gridTemplateColumns: { xs: "1fr", sm: "repeat(2, minmax(0, 1fr))" }`。
- 選択肢カード色は `primary.main`。
- `選択肢` ラベルは選択肢が 1 件以上あるときだけ表示する。

### 4. 確認観点
- メッセージ + 選択肢が画像内に収まり、右端欠けが出ない。
- `xs` は 1 列、 `sm` 以上は 2 列。
- 選択肢ボタン色が `primary.main`。
- 選択肢 0 件のとき、 `選択肢` ラベルが表示されない。
- 選択肢クリックでダイアログ遷移が従来通り。

## レビュー結果 (統合)
- 第 1 段階レビュー: 軽めレビュー 2 点を反映し LGTM。
- 第 2 段階レビュー: オーバーレイ境界指定と `maxWidth` 数値具体化を反映し LGTM。

## 実装結果 (統合)
- 選択肢を左側へ移動し、最終的に画像オーバーレイ内のメッセージ直下へ配置。
- オーバーレイ境界を `left/right/bottom` + `boxSizing: border-box` で固定。
- 全体幅を `maxWidth: 960` に調整。
- 選択肢カード色を `primary.main` に統一。
- 選択肢が空のとき `選択肢` ラベルを非表示化。
- 検証: `docker compose exec react-service yarn lint` 成功。
